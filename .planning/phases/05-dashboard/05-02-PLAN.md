<objective>
Build the two core pages: market overview (heatmap) and ticker detail (candlestick charts + indicators + AI verdict).

Purpose: These two pages deliver DASH-01, DASH-02, DASH-04, and DASH-06 — the core visual experience.
Output: Fully functional market heatmap and interactive ticker detail page with charts and AI analysis.
</objective>

<tasks>
## Task 1: Market Overview Page (Home)

### 1.1: Create heatmap component
Create `frontend/src/components/heatmap.tsx`:
- CSS Grid layout, cells sized proportional to market_cap (or equal sizing if simpler)
- Color scale: deep red (-5%+) → light red → gray (0%) → light green → deep green (+5%+)
- Each cell shows: symbol, daily change %
- Grouped by sector (section headers)
- onClick navigates to `/ticker/${symbol}`
- Note: daily change % needs to be computed from latest price data. Add a helper or new backend endpoint.
  - Simplest: Backend returns latest close and previous close in the ticker list endpoint, or
  - Frontend computes from prices endpoint (but too many API calls for 400 tickers)
  - Decision: Add `last_price`, `prev_close`, `change_pct` fields to GET /tickers response by extending the backend endpoint in this plan

### 1.2: Create ticker search component
Create `frontend/src/components/ticker-search.tsx`:
- Combobox/autocomplete using shadcn Command component
- Filters tickers by symbol or name as user types
- On select, navigates to `/ticker/${symbol}`
- Shows in header/nav area

### 1.3: Build home page
Create `frontend/src/app/page.tsx`:
- Header with logo "Holo", search bar, dark mode toggle
- Market stats summary (total tickers, top gainers/losers)
- Heatmap component
- Responsive: on mobile, heatmap cells stack in a scrollable list

## Task 2: Ticker Detail Page

### 2.1: Create candlestick chart component
Create `frontend/src/components/candlestick-chart.tsx`:
- Uses `lightweight-charts` v5 `createChart()` in a `useEffect`
- CandlestickSeries for OHLCV data
- LineSeries overlays for SMA_20, SMA_50, SMA_200 (different colors)
- LineSeries overlays for BB_upper, BB_middle, BB_lower (dotted)
- Volume histogram below price
- Responsive: chart resizes with container via ResizeObserver
- Time range selector: 1M, 3M, 6M, 1Y, 2Y

### 2.2: Create indicator sub-chart component
Create `frontend/src/components/indicator-chart.tsx`:
- RSI pane: line chart with overbought (70) and oversold (30) horizontal lines
- MACD pane: MACD line + signal line + histogram bars
- Uses lightweight-charts or Recharts (whichever is simpler for line data)
- Synced time axis with main candlestick chart

### 2.3: Create analysis card component
Create `frontend/src/components/analysis-card.tsx`:
- Displays AI analysis result for one dimension (technical/fundamental/sentiment/combined)
- Shows: signal badge (colored), score (1-10 bar), reasoning text
- Combined recommendation prominently displayed: MUA/BÁN/GIỮ with confidence
- Vietnamese text formatting

### 2.4: Build ticker detail page
Create `frontend/src/app/ticker/[symbol]/page.tsx`:
- URL: `/ticker/VNM`, `/ticker/FPT`, etc.
- Layout:
  - Header: ticker symbol + name, watchlist add/remove button
  - Main: Candlestick chart (full width)
  - Below chart: RSI + MACD indicator panes
  - Side/below: Analysis cards (technical, fundamental, sentiment)
  - Bottom: Combined recommendation card (prominent)
- All data fetched via React Query hooks
- Loading states via shadcn Skeleton
- Error states with retry button

### 2.5: Extend backend tickers endpoint for heatmap data
Edit `backend/app/api/tickers.py`:
- Add `last_price`, `prev_close`, `change_pct` to the ticker list response
- Join with daily_prices table to get latest 2 days of prices
- This avoids 400 individual API calls from the frontend
</tasks>
