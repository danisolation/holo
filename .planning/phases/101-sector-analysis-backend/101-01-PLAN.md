  <behavior>
    - get_sector_performance(start_date, end_date) returns list of dicts with sector name, ticker_count, avg_change_today, avg_change_7d, avg_change_30d
    - Tickers with sector=None or sector="" are grouped as "Khác"
    - get_sector_flow(start_date, end_date) returns list of dicts with sector, date, net_volume (sum of volume * sign(close - prev_close))
    - Empty date range returns empty list
    - Sectors with only 1 ticker still work correctly
  </behavior>
  <action>
1. Create `backend/app/schemas/sector.py` with Pydantic models:
   - `SectorPerformanceItem(BaseModel)`: sector (str), ticker_count (int), avg_change_today (float | None), avg_change_7d (float | None), avg_change_30d (float | None)
   - `SectorFlowItem(BaseModel)`: sector (str), date (str), net_volume (float), buy_volume (float), sell_volume (float)

2. Create `backend/app/services/sector_analysis_service.py`:
   - Constructor takes `AsyncSession`
   - `async def get_sector_performance(self, start_date: date, end_date: date) -> list[dict]`:
     - SQL: JOIN DailyPrice with Ticker on ticker_id
     - Use window function LAG(close) OVER (PARTITION BY ticker_id ORDER BY date) to get prev_close
     - Compute pct_change = (close - prev_close) / prev_close * 100
     - For 7D/30D: use LAG(close, 5) and LAG(close, 22) respectively (trading days)
     - GROUP BY COALESCE(Ticker.sector, 'Khác')
     - SELECT sector, COUNT(DISTINCT ticker_id), AVG(pct_change) for each period
     - Filter by date range on DailyPrice.date
   - `async def get_sector_flow(self, start_date: date, end_date: date) -> list[dict]`:
     - SQL: JOIN DailyPrice with Ticker, use LAG for prev_close
     - net_volume = SUM(volume * CASE WHEN close > prev_close THEN 1 WHEN close < prev_close THEN -1 ELSE 0 END)
     - Also compute buy_volume (close > prev_close) and sell_volume (close < prev_close) separately
     - GROUP BY COALESCE(Ticker.sector, 'Khác'), date
     - Filter by date range, ORDER BY date, sector
   - Use `func.coalesce(Ticker.sector, 'Khác')` for null sector handling
   - Use SQLAlchemy `select()`, `join()`, `func.lag()` with `over()` — wrap in CTE/subquery for the LAG window function since you can't GROUP BY a window result directly

3. Create `backend/tests/test_sector_analysis_service.py`:
   - Test get_sector_performance returns correct structure
   - Test null sector mapped to "Khác"
   - Test get_sector_flow returns net_volume with correct sign
   - Test empty result for future date range
   - Use mock_db_session fixture, mock execute().fetchall() returns
  </action>
