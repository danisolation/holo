**Important mocking patterns to follow:**

For `_get_run` calls that happen inside each method, use `patch.object(BacktestAnalyticsService, '_get_run')` as an AsyncMock returning a mock run object. This avoids having to mock the first session.execute call in every test.

For the session.execute mock that handles SQLAlchemy queries, the pattern is:
```python
mock_session = AsyncMock()
# For aggregate queries returning a single row:
mock_result = MagicMock()
mock_row = MagicMock()
mock_row.total = 5
mock_row.wins = 3
mock_row.total_pnl = Decimal("500000")
mock_result.one.return_value = mock_row
mock_session.execute.return_value = mock_result

# For queries returning multiple rows:
mock_result = MagicMock()
mock_result.all.return_value = [mock_row1, mock_row2, ...]
mock_session.execute.return_value = mock_result

# For scalars().all() pattern:
mock_result = MagicMock()
mock_result.scalars.return_value.all.return_value = [equity1, equity2, ...]
mock_session.execute.return_value = mock_result
```

For methods that call session.execute multiple times, use `mock_session.execute.side_effect = [result1, result2, ...]`.
