**New indicator line colors (this phase):**

| Indicator | Color | Hex | lineWidth |
|-----------|-------|-----|-----------|
| ATR line | Amber | `#FBBF24` | 2 |
| ADX line | Cyan | `#06B6D4` | 2 |
| +DI line | Green | `#22C55E` | 1 |
| -DI line | Red | `#EF4444` | 1 |
| Stochastic %K | Pink | `#EC4899` | 2 |
| Stochastic %D | Indigo | `#818CF8` | 1 |

**Reference line colors (this phase):**

| Reference | Color | lineStyle |
|-----------|-------|-----------|
| ADX 25 (strong trend) | `rgba(255,255,255,0.25)` | 2 (dashed) |

| Stoch 20 (oversold) | `rgba(38,166,154,0.4)` | 2 (dashed) |

Color rationale: ATR amber signals "caution/volatility". ADX cyan is spectrally distinct from MACD blue (#2196F3). +DI/#22C55E and -DI/#EF4444 use standard bull/bear green/red. Stochastic %K pink and %D indigo are unique across all charts. Overbought/oversold reuse the RSI zone colors for consistency.

Accent reserved for: N/A this phase — no interactive accent-colored elements are added. Chart colors are data-visualization only.

## Layout & Interaction Contract

### Accordion Behavior

- Accordion `type="multiple"` — multiple charts can be open simultaneously
- Each AccordionItem has a trigger showing the chart name + inline color legend (matching existing h4 pattern)
- Accordion trigger inherits: `text-xs font-medium text-muted-foreground`

**Default expanded state:**

| Chart | Default | Rationale |
|-------|---------|-----------|
| RSI (14) | expanded | Existing behavior preserved — was always visible |
| MACD (12, 26, 9) | expanded | Existing behavior preserved — was always visible |
| ATR (14) | collapsed | New indicator, opt-in to avoid page bloat |
| ADX (14) | collapsed | New indicator, opt-in to avoid page bloat |
| Stochastic (14, 3) | collapsed | New indicator, opt-in to avoid page bloat |

Source: CONTEXT.md Agent's Discretion — "Accordion default state"

### Sub-Chart Dimensions

| Property | Value | Source |
|----------|-------|--------|
| Height | 160px | CONTEXT.md: "match existing 160px from RSI chart" |
| Width | 100% container (ResizeObserver) | Existing pattern |
| Border radius | `rounded-lg` on chart container | Existing pattern |
| Overflow | `overflow-hidden` on chart container | Existing pattern |

### Chart Configuration (all 3 new charts)

```typescript
{
  layout: {
    background: { color: "#0f172a" },
    textColor: "#cbd5e1",
  },
  grid: {
    vertLines: { color: "#1e293b" },
    horzLines: { color: "#1e293b" },
  },
  width: container.clientWidth,
  height: 160,
  rightPriceScale: {
    borderColor: "#334155",
    scaleMargins: { top: 0.1, bottom: 0.1 },

  timeScale: {
    borderColor: "#334155",
    timeVisible: false,
  },
  crosshair: { mode: 0 },

```

### ATR Chart Specifics

- Single line series: ATR value over time
- `priceLineVisible: false`, `lastValueVisible: true`
- No reference lines (ATR is an absolute value, not bounded)
- Right price scale shows ATR in VND

### ADX Chart Specifics

- Three line series: ADX (cyan, lineWidth 2), +DI (green, lineWidth 1), -DI (red, lineWidth 1)
- `lastValueVisible: true` for ADX line, `false` for +DI/-DI
- One reference line at value 25 (dashed white 25% opacity) — strong trend threshold
- Right price scale 0-100

### Stochastic Chart Specifics

- Two line series: %K (pink, lineWidth 2), %D (indigo, lineWidth 1)
- `lastValueVisible: true` for %K, `false` for %D
- Two reference lines: 80 (overbought, dashed red 40%), 20 (oversold, dashed green 40%)
- Right price scale 0-100

## Data Contract (API ↔ Frontend)

### IndicatorData Type Extension

6 new nullable fields added to `IndicatorData` interface in `lib/api.ts`:

```typescript
export interface IndicatorData {
  // ... existing fields ...
  atr_14: number | null;
  adx_14: number | null;
  plus_di_14: number | null;
  minus_di_14: number | null;
  stoch_k_14: number | null;
  stoch_d_14: number | null;
}
```

Field names match database columns from CONTEXT.md decision.

### Data Filtering Pattern

Each chart component filters its data independently (same as RSI/MACD):
- ATR: `indicatorData.filter(d => d.atr_14 != null)`
- ADX: `indicatorData.filter(d => d.adx_14 != null && d.plus_di_14 != null)`
- Stochastic: `indicatorData.filter(d => d.stoch_k_14 != null && d.stoch_d_14 != null)`

Charts render nothing (return `null`) when their filtered data array is empty. The accordion item still appears but shows a flat line or nothing in the chart area.
