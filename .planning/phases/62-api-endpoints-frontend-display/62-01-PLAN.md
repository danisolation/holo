```python
async def _get_ticker_by_symbol(session: AsyncSession, symbol: str) -> Ticker:
    result = await session.execute(select(Ticker).where(Ticker.symbol == symbol.upper()))
    ticker = result.scalar_one_or_none()
    if not ticker:
        raise HTTPException(status_code=404, detail=f"Ticker '{symbol}' not found")
    return ticker
```

**Endpoint 1: `GET /api/rumors/{symbol}`** (per D-2, D-3):
- Resolve symbol → ticker_id using same `_get_ticker_by_symbol` pattern (copy the helper inline or import from analysis — inline is cleaner to avoid cross-import).
- Query `RumorScore` for that ticker_id, ordered by `scored_date DESC`, limit 1 (latest score).
- Query `Rumor` for that ticker_id, ordered by `posted_at DESC`, limit 20 (recent posts for the feed).
- Return `RumorScoreResponse` with score fields from latest RumorScore (or None if no score), and posts list from Rumor rows.
- For `posted_at`, format as ISO string: `row.posted_at.isoformat()`.
- If ticker not found, raise HTTPException 404.
- If no scores AND no posts, return the response with all Nones and empty posts list (NOT 404 — ticker exists but has no rumor data yet).

**Endpoint 2: `GET /api/rumors/watchlist/summary`** (per D-4):
- Query all watchlist symbols from UserWatchlist.
- For each watchlist symbol, compute:
  - `rumor_count`: COUNT of Rumor rows for that ticker in last 7 days
  - `avg_credibility`: AVG of RumorScore.credibility_score in last 7 days
  - `avg_impact`: AVG of RumorScore.impact_score in last 7 days
  - `dominant_direction`: MODE of RumorScore.direction in last 7 days (most frequent)
- Use a single query with JOINs to avoid N+1. Pattern:
  ```python
  from datetime import date, timedelta
  seven_days_ago = date.today() - timedelta(days=7)
  ```
  Join UserWatchlist → Ticker → RumorScore (where scored_date >= seven_days_ago), group by symbol.
  For rumor_count, join separately to Rumor (where posted_at >= seven_days_ago).
- Return `list[WatchlistRumorSummary]`.
- For `dominant_direction`: use a subquery with `func.count` grouped by direction, pick the direction with max count. If no scores, return None.
- NOTE: IMPORTANT — the `/watchlist/summary` route MUST be defined BEFORE `/{symbol}` route in the file, otherwise FastAPI will treat "watchlist" as a {symbol} parameter.

**Register in `backend/app/api/router.py`**:
Add `from app.api.rumors import router as rumors_router` and `api_router.include_router(rumors_router)` — after the discovery_router line.
