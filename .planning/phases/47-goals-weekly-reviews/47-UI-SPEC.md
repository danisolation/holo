## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.6 | `text-sm leading-relaxed` |
| Label | 12px | 400 (regular) | 1.4 | `text-xs` |
| Heading | 24px | 700 (bold) | 1.25 | `text-2xl font-bold tracking-tight` |
| Mono | 14px | 700 (bold) | 1.4 | `font-mono text-sm font-bold` |

Weights: **2 only** — 400 (regular) for body text, labels, secondary copy, review prose + 700 (bold) for headings, mono/numeric values, emphasis.

Notes:
- Monthly goal target value uses `font-mono text-2xl font-bold` (same as performance card values from Phase 45)
- Monthly goal label uses `text-xs text-muted-foreground`
- Progress bar percentage uses `font-mono text-sm font-bold` colored by progress tier
- Weekly prompt question uses `text-sm font-bold` for emphasis
- Weekly prompt buttons use standard Button text (inherits from shadcn)
- Weekly review summary prose uses `text-sm leading-relaxed` (body role) for comfortable reading
- Weekly review highlights/suggestions use `text-sm` in list format
- Goal dialog input label uses `text-sm font-bold`, input uses `font-mono` for VND amounts
- Section headings ("Mục tiêu & Nhận xét") use `text-lg font-bold` (same as Phase 45/46 section headings)

### Progress Bar Color Logic

| Condition | Bar Color | Text Color |
|-----------|-----------|------------|
| progress ≥ 100% | `bg-[#26a69a]` | `text-[#26a69a]` |
| progress 50-99% | `bg-amber-600 dark:bg-amber-400` | `text-amber-600 dark:text-amber-400` |
| progress < 50% | `bg-[#ef5350]` | `text-[#ef5350]` |
| progress = 0 or no trades | `bg-muted` (track only) | `text-muted-foreground` |

Accent reserved for: goal achievement (≥100% progress bar + badge), positive P&L values in weekly review, good habit highlight items.

---

## Visual Focal Point

**Focal point: Monthly goal progress bar → Weekly prompt card (when pending)** — The goal progress bar gives a persistent monthly accountability metric. When a Monday prompt is pending, the weekly risk tolerance card demands an action before the market opens. The weekly review card provides passive coaching insight — expanded by default so the user reads the AI feedback on page load. These sections sit between pick history and behavior insights, creating a natural flow: what happened (picks/history) → how am I doing (goal/review) → what patterns exist (behavior).

#### Monthly Goal Card States

| State | Display |
|-------|---------|
| No goal set | Card shows empty state with CTA button (see Empty States below) |
| Goal set, 0 trades | Progress bar empty (0%), actual = "0 VND", percentage = "0%" in `text-muted-foreground` |
| Goal set, in progress (<100%) | Progress bar filled to percentage, colored by tier |
| Goal set, achieved (≥100%) | Progress bar full (100% width), green fill, "🎉" emoji prefix on percentage: "🎉 120%" |
| Goal set, exceeded (>150%) | Same as achieved but percentage capped at display "200%" max. Bar stays at 100% width |

| Range | Display | Color |

## Number Formatting

| Type | Format | Example | Implementation |
|------|--------|---------|----------------|
| Goal target VND | Thousands separator + " VND" | "5,000,000 VND" | `formatVND(val) + " VND"` |
| Actual P&L VND | Sign prefix + thousands separator | "+2,500,000" / "-300,000" | `(val > 0 ? "+" : "-") + formatVND(Math.abs(val))` |
| Goal percentage | Integer + "%" | "50%" | `Math.round(pct) + "%"` |
| Risk level | Integer + "/5" | "3/5" | `level.toString() + "/5"` |
| Review P&L VND | Sign prefix + thousands separator + " VND" | "+1,200,000 VND" | `(val > 0 ? "+" : "") + formatVND(val) + " VND"` |
| Week date range | DD/MM – DD/MM | "14/04 – 20/04" | Custom: `formatDateVN` shortened (day/month only) |
| Month/Year | M/YYYY | "4/2026" | `(month) + "/" + year` |
| Trade count | Integer + " lệnh" | "5 lệnh" | `count.toString() + " lệnh"` |
| Win count | Integer + " thắng" | "3 thắng" | `count.toString() + " thắng"` |
| Date | DD/MM/YYYY | "23/04/2026" | `formatDateVN()` from `lib/format.ts` |

## Accessibility

| Element | Requirement |
|---------|-------------|
| Progress bar | `role="progressbar"` + `aria-valuenow={pct}` + `aria-valuemin={0}` + `aria-valuemax={100}` + `aria-label="Tiến độ mục tiêu tháng"` |
| Goal card values | Actual P&L sign (+/-) always present in text — never color-only |
| Goal percentage | Text always present — never communicated by bar width alone |
| Set goal dialog | Focus trapped inside dialog when open. `aria-labelledby` pointing to DialogTitle |
| Goal input | `aria-describedby` linking to validation error message (when present) |
| Weekly prompt buttons | Clear text labels with icons — icons have `aria-hidden="true"`. Text carries meaning |
| Weekly prompt card | `role="region"` + `aria-label="Khảo sát rủi ro hàng tuần"` |
| Prompt button disabled state | `title` attribute explains why disabled ("Đã ở mức thận trọng nhất") |
| Review card toggle | `button` role on header with `aria-expanded="true/false"` + `aria-controls="review-content"` |
| Review content | `id="review-content"` for aria-controls reference |
| Review highlights | `ul` semantic list elements. Emoji icons are decorative (`aria-hidden="true"` on emoji spans) |
| Review stats footer | `sr-only` label: "Thống kê tuần" before stats row |
| Error retry buttons | Focus returned to retry button after failed refetch attempt |

## Data Flow

| Hook | API Endpoint | staleTime | Usage |
|------|-------------|-----------|-------|
| `useCurrentGoal()` | `GET /api/goals/current` | 5 min | Monthly goal card (target, actual, progress) |
| `useSetGoal()` | `POST /api/goals` | mutation | Set/update monthly target. Invalidates `["goals", "current"]` |
| `useWeeklyPrompt()` | `GET /api/goals/weekly-prompt` | 5 min | Weekly prompt card (conditional render) |
| `useRespondWeeklyPrompt()` | `POST /api/goals/weekly-prompt/{id}/respond` | mutation | Submit prompt response. Invalidates `["goals", "weekly-prompt"]` + `["profile"]` |
| `useLatestReview()` | `GET /api/goals/weekly-review` | 10 min | Latest weekly review card |
