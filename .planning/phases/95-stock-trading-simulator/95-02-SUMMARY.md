# Phase 95 Plan 02: Simulator Backend Foundation Summary

**One-liner:** Paper trading backend with 3 DB models, FIFO P&L matching, VN market fee calculation (0.15% buy + 0.25% sell), and 5 REST API endpoints.

## What Was Built

### Task 1: Simulator Models & Alembic Migration
- **SimulatorPortfolio**: Virtual portfolio with 100M VND starting capital, tracks current cash
- **SimulatorTrade**: Paper trades with source tracking (ai_auto/manual), fee breakdown, P&L fields
- **SimulatorLot**: FIFO inventory lots for position tracking with remaining_quantity
- Alembic migration `273292a3289a` creates all 3 tables with proper FK constraints
- Models registered in `__init__.py` with `__all__` export

### Task 2: Service Layer & API Endpoints
- **Schemas** (Pydantic): `SimulatorTradeCreate` with regex validation for side/source, `PortfolioResponse` with positions, `SimulatorStatsResponse` for AI vs manual comparison
- **SimulatorService**: Fee calculation (0.15% broker buy, 0.15% broker + 0.1% tax sell), FIFO lot matching on sell, portfolio state with unrealized P&L from latest daily prices
- **5 API Endpoints**:
  - `GET /api/simulator/portfolio` — portfolio state with positions
  - `POST /api/simulator/trades` — create paper trade
  - `GET /api/simulator/trades` — paginated trade history (filterable by source)
  - `GET /api/simulator/stats` — AI vs manual performance comparison
  - `POST /api/simulator/reset` — reset portfolio to starting capital
- Router registered in `backend/app/api/router.py`
