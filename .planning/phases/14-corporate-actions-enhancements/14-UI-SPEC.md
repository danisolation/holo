# Phase 14 — UI Design Contract

> Visual and interaction contract for Corporate Actions Enhancements. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | base-nova |
| Component library | base-ui (via shadcn v4) |
| Icon library | lucide-react |
| Font | Geist Sans (`--font-sans`), Geist Mono (`--font-geist-mono`) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, calendar cell inner padding |
| sm | 8px | Compact element spacing, calendar grid gap, popover item gap |
| md | 16px | Default element spacing, card padding, calendar section padding |
| lg | 24px | Section padding, calendar card header spacing |
| xl | 32px | Layout gaps between dashboard sections |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: Calendar day cells use 40×40px minimum touch target for date numbers.

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Popover event details, calendar event labels |
| Label | 12px | 400 | 1.5 | Day-of-week headers, event count badges, muted captions |
| Heading | 18px | 600 | 1.2 | Section title "Lịch sự kiện doanh nghiệp" |
| Display | 24px | 600 | 1.2 | Page heading (reuse existing `text-2xl font-bold`) |

Additional rules:
- Numeric data (prices, percentages, ratios) always use `font-mono` (Geist Mono)
- Day numbers in calendar cells: 14px weight 400
- Month/year label in calendar header: 18px weight 600

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / `oklch(0.145 0 0)` dark | Page background, calendar grid background |
| Secondary (30%) | `var(--card)` / `oklch(0.205 0 0)` dark | Calendar card, popover surface, toggle inactive |
| Accent (10%) | `var(--primary)` | Active toggle state ("Giá điều chỉnh"/"Giá gốc" selected), calendar "today" ring |
| Destructive | `var(--destructive)` | Not used this phase (no destructive actions) |

## Component Inventory

### 1. EventCalendar

**Location:** `frontend/src/components/event-calendar.tsx`
**Type:** Client component (`"use client"`)
**Wrapping:** `<Card>` with `<CardHeader>` + `<CardContent>`

**Layout:**
- Card title: "Lịch sự kiện doanh nghiệp" (18px, weight 600)
- CardAction slot: month navigation — `<` prev month, current "Tháng MM/YYYY" label, `>` next month
- Below header: EventCalendarFilters component
- Calendar grid: 7-column CSS grid (`grid-cols-7`)
- Row 1: Day-of-week headers (T2, T3, T4, T5, T6, T7, CN) — 12px, weight 400, muted-foreground, text-center
- Rows 2+: Day cells — 6 rows max

**Day Cell Spec:**
- Min size: 40×40px (touch target)
- Background: transparent default, `var(--muted)` on hover
- Border-radius: `var(--radius-md)` (0.5rem)
- Day number: 14px, weight 400, top-left aligned with 4px padding
- Today indicator: 2px ring using `var(--primary)`, `ring-2 ring-primary`
- Days outside current month: `opacity-40`
- Event dots: Row of colored 4px circles below the day number, max 4 visible, `+N` text if more
- Click handler: opens EventDayPopover anchored to the cell
- Cursor: `cursor-pointer` when day has events, `cursor-default` when empty

**State management:** Month/year stored in component-local `useState`. Event data fetched via `useQuery` keyed on `[month, year, eventType]`.

**Empty state:** When no events in the visible month: centered text inside the grid area — "Không có sự kiện trong tháng này"

**Loading state:** Skeleton overlay on the grid body (not the header). Use existing `<Skeleton>` component.

**API:** `GET /api/corporate-events?month=YYYY-MM&type={type}` (type optional, omit for "all")

### 2. EventCalendarFilters

**Location:** `frontend/src/components/event-calendar-filters.tsx`
**Type:** Client component
**Pattern:** Reuse `ExchangeFilter` pattern — `<Tabs>` with `<TabsList>` + `<TabsTrigger>`

**Tabs (line variant):**

| Value | Label | Order |
|-------|-------|-------|
| `all` | Tất cả | 1 |
| `CASH_DIVIDEND` | Cổ tức tiền | 2 |
| `STOCK_DIVIDEND` | Cổ tức CP | 3 |
| `BONUS_SHARES` | Thưởng CP | 4 |
| `RIGHTS_ISSUE` | Quyền mua | 5 |

**Spec:**
- Use `<TabsList variant="line">` for underline style (matches inline filter aesthetic vs. the pill-style exchange filter)
- Tab text: 14px, weight 400
- Active tab: foreground text + bottom underline via the existing TabsTrigger line-variant styling
- Each tab trigger prefixed with a colored dot (6px circle, matching event type color) except "Tất cả"
- Height: `h-9` (matching ExchangeFilter)

**State:** Controlled via prop callback `onTypeChange(type: string)`. Parent (EventCalendar) owns the state.

### 3. EventDayPopover

**Location:** `frontend/src/components/event-day-popover.tsx`
**Type:** Client component
**Requires:** Install `popover` from shadcn: `npx shadcn@latest add popover`

**Trigger:** Calendar day cell click
**Placement:** Bottom-start, with collision-aware flipping

**Popover content spec:**
- Width: `w-72` (288px)
- Max height: `max-h-64` (256px) with `overflow-y-auto`
- Padding: 16px (`p-4`)
- Background: `var(--popover)` (via shadcn Popover defaults)
- Border: `var(--border)` (via shadcn Popover defaults)
- Border-radius: `var(--radius-lg)` (0.625rem)

**Header inside popover:**
- Date label: "Ngày DD/MM/YYYY" — 14px, weight 600
- Event count: Badge variant="secondary" showing count, e.g. "3 sự kiện"
- Separator: `<Separator>` below header (existing shadcn component)

**Event list items:**
- Each item: vertical stack with 8px gap between items
- Event type badge: `<Badge variant="outline">` with event-type color border/text (following ExchangeBadge pattern)
  - CASH_DIVIDEND → "Cổ tức tiền" with `--event-cash-dividend` color
  - STOCK_DIVIDEND → "Cổ tức CP" with `--event-stock-dividend` color
  - BONUS_SHARES → "Thưởng CP" with `--event-bonus-shares` color
  - RIGHTS_ISSUE → "Quyền mua" with `--event-rights-issue` color
- Symbol: `font-mono font-bold` 14px — clickable link to `/ticker/{symbol}`
- Detail line 1: "Ex-date: DD/MM/YYYY" — 12px, muted-foreground
- Detail line 2 (dividend): "Tỷ lệ: {ratio} | Giá trị: {amount} VNĐ/CP" — 12px, muted-foreground, font-mono for numbers
- Detail line 2 (rights): "Tỷ lệ: {ratio} | Giá: {price} VNĐ/CP" — 12px, muted-foreground, font-mono for numbers

**Empty state (day with no events — shouldn't normally render but defensive):**
- "Không có sự kiện" — 12px, muted-foreground, centered

### 4. PriceToggle

**Location:** Inline within existing `candlestick-chart.tsx` (not a separate file)
**Type:** Part of existing client component

**Placement:** In the chart header row, after the time range buttons, separated by a `<Separator orientation="vertical" className="h-4" />`

**Spec:**
- Two `<Button>` elements in a group with `gap-0` and shared border:
  - Left button: "Giá ĐC" (short for Giá điều chỉnh = Adjusted)
  - Right button: "Giá gốc" (Raw)
- Active button: `variant="default"` (filled primary)
- Inactive button: `variant="outline"`
- Size: `size="xs"` (matching existing time range buttons)
- Left button: `rounded-r-none border-r-0`
- Right button: `rounded-l-none`
- Muted prefix label: "Hiển thị:" — 12px, muted-foreground, matching "Khoảng thời gian:" pattern

**State:** `useState<"adjusted" | "raw">("adjusted")` — local to CandlestickChart component. Default: `"adjusted"`.

**Behavior:**
- When toggled to "raw": parent refetches price data with `?adjusted=false` query param
- When toggled to "adjusted": parent refetches with `?adjusted=true` (or omit, as default)
- New prop on CandlestickChart: `adjusted: boolean` + `onAdjustedChange: (adjusted: boolean) => void`
- Parent (ticker detail page) owns the fetch logic; CandlestickChart just renders toggle + calls back

**Data mapping:**
- adjusted=true: uses `adjusted_close` field (existing behavior, `d.close` already mapped)
- adjusted=false: uses `close` field from raw price endpoint

### 5. DilutionBadge

**Location:** `frontend/src/components/dilution-badge.tsx`
**Type:** Server-compatible component (no client hooks needed)

**Placement:** Rendered inline in the HoldingsTable "Mã CK" (symbol) column, next to the ticker symbol. Only shown when a rights issue affects that ticker.

**Visual spec:**
- Uses `<Badge variant="outline">` with amber event-type color
- Border: `border-[var(--event-rights-issue)]`
- Text: `text-[var(--event-rights-issue)]` (amber)
- Background: `bg-[var(--event-rights-issue)]/10` (10% opacity amber)
- Icon: `AlertTriangle` from lucide-react, `size-3`, inline before text
- Text: "Pha loãng {X}%" — 12px, font-mono for the percentage
- Height: `h-5` (matching existing Badge)

**Tooltip (on hover):**
- Use native `title` attribute (no tooltip component needed for single line)
- Content: "Quyền mua {ratio} — Ex-date: DD/MM/YYYY"

**Props:**
```typescript
interface DilutionBadgeProps {
  dilutionPct: number;    // e.g. 15.5
  ratio: string;          // e.g. "10:3"
  exDate: string;         // e.g. "2026-05-15"
}
```

**Behavior:**
- Only rendered when backend returns `rights_issue` data for a held ticker
- Positioned to the right of ticker symbol with `gap-2` in a flex row
- Badge is non-interactive (no click action)

## Interaction Patterns

### Calendar Navigation
- Left/Right arrow buttons: `<Button variant="ghost" size="xs">` with `<ChevronLeft>` / `<ChevronRight>` icons
- Click advances/retreats by 1 month
- No keyboard shortcut for month navigation (mouse-primary interaction)
- Button disabled state: none (allow browsing any month)

### Calendar Day Click
1. User clicks day cell with events → EventDayPopover opens
2. User clicks day cell without events → no action (cursor stays default)
3. User clicks outside popover → popover closes
4. User clicks different day → current popover closes, new one opens
5. Only one popover open at a time

### Price Toggle
1. User clicks inactive toggle button → active state swaps
2. Chart shows brief loading indicator (existing skeleton or opacity transition)
3. New price data loads → chart re-renders with new OHLCV data
4. Toggle state persists only within the component lifecycle (reset on page navigation)

### DilutionBadge
1. Static display — no click interaction
2. Hover shows native tooltip with rights issue details
3. Badge only appears when backend includes `rights_issue` in holdings response
