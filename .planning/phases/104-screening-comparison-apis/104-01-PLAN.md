```python
from pydantic import BaseModel

class ScreenerTickerItem(BaseModel):
    """Single ticker in screener results."""
    symbol: str
    name: str
    sector: str | None = None
    industry: str | None = None
    close: float | None = None
    volume: int | None = None
    change_1d: float | None = None   # % change vs prev close
    change_7d: float | None = None
    change_30d: float | None = None
    pe: float | None = None          # from Financial table, nullable
    market_cap: float | None = None

class ScreenerResponse(BaseModel):
    """Paginated screener results."""
    items: list[ScreenerTickerItem]
    total: int
    offset: int
    limit: int

class PeerComparisonItem(BaseModel):
    """Single peer in comparison."""
    symbol: str
    name: str
    close: float | None = None
    volume: int | None = None
    change_1d: float | None = None
    pe: float | None = None
    market_cap: float | None = None
    rank_pe: int | None = None
    rank_volume: int | None = None
    rank_change: int | None = None
    rank_market_cap: int | None = None
    is_target: bool = False  # True for the queried ticker

class PeerComparisonResponse(BaseModel):
    """Peer comparison results."""
    symbol: str
    sector: str
    peers: list[PeerComparisonItem]

class SectorDetailTickerItem(BaseModel):
    """Single ticker in sector detail."""
    symbol: str
    name: str
    industry: str | None = None
    close: float | None = None
    volume: int | None = None
    change_7d: float | None = None
    change_30d: float | None = None
    market_cap: float | None = None

class SectorDetailResponse(BaseModel):
    """Sector detail results."""
    sector: str
    ticker_count: int
    tickers: list[SectorDetailTickerItem]
```

**2. Create `backend/app/services/screener_service.py`:**

Follow the SectorAnalysisService pattern (constructor takes AsyncSession, methods use SQLAlchemy async queries).

Implement class `ScreenerService`:

**`screen_tickers(sector, industry, min_volume, max_volume, min_change, max_change, min_pe, max_pe, sort_by, sort_order, limit, offset)`:**
- Query: JOIN Ticker + latest DailyPrice (subquery for max date per ticker) + LEFT JOIN Financial (latest period per ticker using subquery with `ORDER BY year DESC, quarter DESC NULLS LAST LIMIT 1` or `DISTINCT ON`)
- Use a subquery/CTE to get the latest DailyPrice row per ticker (max date)
- Compute change_1d from LAG(close,1) or just use (close - prev_day_close)/prev_day_close. Simpler approach: get the latest 2 prices per ticker for 1D change, latest price vs price ~5 trading days ago for 7D, ~22 days for 30D.
- **Simpler approach for screener** (avoids heavy LAG over full history): Use a CTE that fetches the latest price date, then the price N days back, compute % changes.
- Apply WHERE filters: `Ticker.sector == sector` (if provided), `Ticker.industry == industry` (if provided), volume >= min_volume, volume <= max_volume, change_1d >= min_change, change_1d <= max_change, pe >= min_pe, pe <= max_pe.
- Apply ORDER BY based on sort_by param (map to column: "volume", "change_1d", "change_7d", "change_30d", "pe", "close", "market_cap"). Default sort_by="volume", sort_order="desc".
- Apply LIMIT/OFFSET. Also compute total count for pagination.
- Return dict with items list, total, offset, limit.

**`get_peer_comparison(symbol)`:**
- Look up the ticker's sector. If no sector, raise or return empty.
- Get all active tickers in the same sector.
- For each: latest close, latest volume, 1D % change, latest P/E from Financial, market_cap from Ticker.
- Rank each metric using Python (sorted lists, assign rank 1=best). Rank P/E ascending (lower=better), volume descending (higher=better), change_1d descending (higher=better), market_cap descending (higher=better).
- Mark the target ticker with `is_target=True`.
- Return dict with symbol, sector, peers list.

**`get_sector_detail(sector_name)`:**
- Get all active tickers WHERE sector = sector_name.
- For each ticker: latest close, latest volume, 7D change, 30D change, market_cap.
- Use same price/change computation approach as screener.
- Return dict with sector, ticker_count, tickers list.

**Performance notes:**
- For change computation, use a helper method `_get_prices_for_tickers(ticker_ids, lookback_days=30)` that fetches prices for the last 30 days for given tickers, then compute changes in Python. This avoids complex LAG CTEs across full history.
- Use `func.max(DailyPrice.date)` subquery to find latest date, then join back for the actual price row.
- For the P/E join, use a lateral subquery or a simple subquery: `SELECT pe FROM financials WHERE ticker_id = t.id ORDER BY year DESC, quarter DESC NULLS LAST LIMIT 1`.

**Important:** All filter params are Optional[None]. Only apply WHERE clause for non-None params. P/E comes from the `Financial` model (field `pe`), NOT from DailyPrice. If no Financial record exists for a ticker, pe = None.
