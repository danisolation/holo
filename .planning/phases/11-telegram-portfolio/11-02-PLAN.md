**Add portfolio data fetching** at the END of `build_daily_summary()`, just before `return data` (after the existing "new strong recommendations" block, around line 317):

```python
# --- Portfolio P&L (TBOT-04 + TBOT-06) ---
try:
    portfolio_svc = PortfolioService(self.session)
    holdings = await portfolio_svc.get_holdings()
    if holdings:
        data["portfolio_holdings"] = holdings
        portfolio_summary = await portfolio_svc.get_summary()
        data["portfolio_summary"] = portfolio_summary
        # Owned symbols set for TBOT-06 sorting in formatter
        data["owned_symbols"] = {h["symbol"] for h in holdings}
except Exception as e:
    logger.warning(f"Failed to fetch portfolio data for daily summary: {e}")
    # Non-critical — summary sends without portfolio section
```

**Key design decisions:**
- Wrap in try/except because portfolio data is a non-critical enhancement — if it fails, the summary still sends with market data (graceful degradation)
- Use the SAME session (`self.session`) — PortfolioService accepts session in constructor, no new connection needed
- `owned_symbols` is a set for O(1) lookup in the formatter's sorting logic
- `portfolio_holdings` and `portfolio_summary` are only added to dict if user has positions (per must-have: "no empty block")

**1. Add portfolio P&L section** — insert AFTER the "Top movers" block and BEFORE "Watchlist signal changes" (between lines ~181 and ~183 in current code). Only render if `data.get("portfolio_holdings")` is truthy:

```python
# Portfolio P&L section (TBOT-04)
portfolio = data.get("portfolio_holdings", [])
portfolio_summary = data.get("portfolio_summary", {})
if portfolio:
    lines.append("💼 <b>Danh mục của bạn:</b>")
    for h in portfolio:
        pnl = h.get("unrealized_pnl")
        pnl_pct = h.get("unrealized_pnl_pct")
        emoji = "🟢" if (pnl or 0) >= 0 else "🔴"
        price_str = f"{h['market_price']:,.0f}đ" if h.get("market_price") else "—"
        pnl_str = f"{pnl:+,.0f}đ ({pnl_pct:+.1f}%)" if pnl is not None else "—"
        lines.append(f"  {emoji} <b>{h['symbol']}</b>: {price_str} | P&L: {pnl_str}")

    # Portfolio totals
    total_upnl = portfolio_summary.get("total_unrealized_pnl")
    if total_upnl is not None:
        total_emoji = "📈" if total_upnl >= 0 else "📉"
        lines.append(f"  {total_emoji} Tổng P&L: {total_upnl:+,.0f}đ")
        return_pct = portfolio_summary.get("total_return_pct")
        if return_pct is not None:
            lines.append(f"  📊 Lợi nhuận: {return_pct:+.1f}%")
    lines.append("")
```

**2. Sort recommendations by owned-first** — in the "New strong recommendations" block (around line ~196), modify the rendering to sort owned tickers to the top. Per D-11-04: position-specific P&L annotation.

Replace the recommendations rendering block with:
```python
recs = data.get("new_recommendations", [])
if recs:
    owned_symbols = data.get("owned_symbols", set())
    # Sort: owned tickers first, then by score descending (original order)
    owned_recs = [r for r in recs if r.get("symbol") in owned_symbols]
    other_recs = [r for r in recs if r.get("symbol") not in owned_symbols]
    sorted_recs = owned_recs + other_recs

    lines.append("💡 <b>Khuyến nghị mới:</b>")
    for r in sorted_recs[:5]:
        emoji = MessageFormatter._recommendation_emoji(r.get("signal", ""))
        symbol = r.get("symbol", "?")
        # Add position annotation for owned tickers (TBOT-06)
        owned_tag = " 📌" if symbol in owned_symbols else ""
        lines.append(
            f"  {emoji} <b>{symbol}</b>{owned_tag}: "
            f"{r.get('signal', '—')} ({r.get('score', '—')}/10)"
        )
    lines.append("")
```

The 📌 emoji annotates tickers the user owns, making them visually distinct.
