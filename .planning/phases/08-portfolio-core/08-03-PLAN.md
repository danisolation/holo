**Endpoint 1: `POST /trades`** — per PORT-01
```python
@router.post("/trades", response_model=TradeResponse, status_code=201)
async def create_trade(trade: TradeRequest):
    """Record a buy or sell trade."""
    async with async_session() as session:
        service = PortfolioService(session)
        try:
            result = await service.record_trade(
                symbol=trade.symbol,
                side=trade.side,
                quantity=trade.quantity,
                price=trade.price,
                trade_date=trade.trade_date,
                fees=trade.fees,
            )
            return TradeResponse(**result)
        except ValueError as e:
            error_msg = str(e)
            if "not found" in error_msg:
                raise HTTPException(status_code=404, detail=error_msg)
            raise HTTPException(status_code=400, detail=error_msg)
```

**Endpoint 2: `GET /holdings`** — per PORT-02, PORT-05

**Endpoint 4: `GET /trades`** — per PORT-07
```python
@router.get("/trades", response_model=TradeHistoryResponse)
async def get_trades(
    ticker: str | None = Query(None, description="Filter by ticker symbol"),
    side: str | None = Query(None, pattern="^(BUY|SELL)$", description="Filter by side"),
    limit: int = Query(50, ge=1, le=200),
    offset: int = Query(0, ge=0),
):
    """Get trade history with optional filtering."""
    async with async_session() as session:
        service = PortfolioService(session)
        result = await service.get_trades(
            ticker_symbol=ticker, side=side, limit=limit, offset=offset
        )
        return TradeHistoryResponse(
            trades=[TradeResponse(**t) for t in result["trades"]],
            total=result["total"],
        )
```

CRITICAL pattern: Use `async with async_session() as session:` — NOT `Depends(get_db)`. This is the established project convention (verified in tickers.py, analysis.py).

**Class: TestFIFO** (PORT-03)

- `test_fifo_partial_consumption` — 1 lot of 200, sell 150, verify remaining = 50

**Class: TestRealizedPnL** (PORT-04)

- `test_realized_pnl_multi_lot` — 2 lots different prices, partial sell, verify weighted P&L

**Class: TestSellValidation** (PORT-01, D-08-06)
- `test_sell_exceeds_available_raises_error` — verify ValueError message
- `test_sell_exact_amount_succeeds` — boundary case
- `test_sell_zero_holdings_raises_error` — no lots exist

**Class: TestTradeRecording** (PORT-01)
- `test_buy_creates_trade_and_lot` — verify Trade + Lot objects added to session
- `test_sell_creates_trade_and_consumes_lots` — verify Trade added, lots consumed
- `test_unknown_ticker_raises_error` — verify ValueError("Ticker 'XYZ' not found")

**Implementation approach:**
Since the service uses SQLAlchemy queries internally, tests need to mock `session.execute()` returns. For each test:
1. Set up mock session with `execute` returning mock result objects
2. Mock `result.scalars().all()` for list queries (lots), `result.scalar_one_or_none()` for single queries (ticker lookup), `result.scalar_one()` for aggregate queries (sum)
3. Create real `Lot` objects (or mock objects with the right attributes) for FIFO tests
4. Call service method and assert return values / side effects

For FIFO-specific tests, create mock Lot objects with `remaining_quantity` attributes and verify they're modified correctly after `_consume_lots_fifo` is called.

**Alternative simpler approach:** Test the `_consume_lots_fifo` method more directly by creating mock lots and verifying the algorithm. Use `patch` to mock the session.execute call within the method.

The test file should have at minimum 10 test functions covering the behaviors listed above. Each test should be focused and test one behavior.
