    - SELL trade: validates sell_qty <= available shares (sum of remaining_quantity for ticker), consumes lots FIFO (oldest buy_date first, then id), returns Trade with realized_pnl
    - FIFO partial consumption: selling 150 shares against lots [100, 100] → lot1.remaining=0, lot2.remaining=50
    - FIFO full consumption: selling 100 against lot [100] → lot.remaining=0
    - Sell validation fails: selling 200 against 150 available → raises ValueError("Cannot sell 200 shares — only 150 available")
    - Realized P&L: sell 100@120 against lot bought at 100 with sell_fees=50 → (120-100)*100 - 50 = 1950
    - Symbol resolution: unknown symbol → raises ValueError("Ticker 'XYZ' not found")
    - Fee handling per D-08-07: fees subtracted from P&L, not cost basis. Buy fees tracked per trade. Sell-side: realized_pnl -= sell_fees
  </behavior>

"""Portfolio service: trade recording, FIFO lot management, P&L computation.

Core business logic for Phase 8 portfolio tracking.
Per D-08-01/02/03/06/07 from CONTEXT.md.

from datetime import date
from decimal import Decimal
from sqlalchemy import select, func
from sqlalchemy.ext.asyncio import AsyncSession
from loguru import logger
from app.models.ticker import Ticker
from app.models.trade import Trade
from app.models.lot import Lot

**Method: `record_trade(symbol, side, quantity, price, trade_date, fees=0)`**
1. Resolve symbol → ticker_id via `select(Ticker).where(Ticker.symbol == symbol.upper())`. Raise `ValueError` if not found. (Follows pattern from analysis.py `_get_ticker_by_symbol`.)
2. Convert price and fees to `Decimal` (from float input): `Decimal(str(price))`, `Decimal(str(fees))`
3. Create Trade row: `Trade(ticker_id=..., side=side, quantity=quantity, price=price_dec, fees=fees_dec, trade_date=trade_date)`
4. `session.add(trade)` then `await session.flush()` to get trade.id
5. **If BUY**: Create Lot: `Lot(trade_id=trade.id, ticker_id=ticker_id, buy_price=price_dec, quantity=quantity, remaining_quantity=quantity, buy_date=trade_date)`. `session.add(lot)`.
6. **If SELL**: Call `_consume_lots_fifo(ticker_id, quantity, price_dec, fees_dec)` → returns `Decimal` realized_pnl.
7. `await session.commit()`
8. Return dict: `{trade_id, symbol, side, quantity, price, fees, trade_date, created_at, realized_pnl}`

**Method: `_validate_sell(ticker_id, sell_qty)`** — per D-08-06
1. `select(func.coalesce(func.sum(Lot.remaining_quantity), 0)).where(Lot.ticker_id == ticker_id, Lot.remaining_quantity > 0)`
2. If `sell_qty > available`: raise `ValueError(f"Cannot sell {sell_qty} shares — only {available} available")`

**Method: `_consume_lots_fifo(ticker_id, sell_qty, sell_price, sell_fees)`** — per D-08-03
1. Call `_validate_sell(ticker_id, sell_qty)` first
2. Query open lots: `select(Lot).where(Lot.ticker_id == ticker_id, Lot.remaining_quantity > 0).order_by(Lot.buy_date.asc(), Lot.id.asc())`
3. Iterate lots, tracking `remaining_sell = sell_qty` and `realized_pnl = Decimal("0")`:
   - `consumed = min(lot.remaining_quantity, remaining_sell)`
   - `realized_pnl += (sell_price - lot.buy_price) * consumed`
   - `lot.remaining_quantity -= consumed`
   - `remaining_sell -= consumed`
   - Break when `remaining_sell <= 0`
4. Subtract sell fees: `realized_pnl -= sell_fees`
5. Return `realized_pnl`

CRITICAL: All money arithmetic uses `Decimal`. Never use `float` for intermediate calculations. Convert float inputs via `Decimal(str(value))` to avoid IEEE 754 precision issues.

**Method: `get_holdings()` → list[dict]** — per PORT-02, PORT-05

   - Use SQLAlchemy: `select(Lot.ticker_id, func.sum(Lot.remaining_quantity).label("total_qty"), (func.sum(Lot.buy_price * Lot.remaining_quantity) / func.sum(Lot.remaining_quantity)).label("avg_cost"), func.sum(Lot.buy_price * Lot.remaining_quantity).label("total_cost")).where(Lot.remaining_quantity > 0).group_by(Lot.ticker_id)`
2. For each ticker with open lots, get latest daily_prices.close using a subquery:
   - Subquery: `select(DailyPrice.ticker_id, DailyPrice.close).distinct(DailyPrice.ticker_id).order_by(DailyPrice.ticker_id, DailyPrice.date.desc())` or use ROW_NUMBER pattern from tickers.py market_overview endpoint.
   - Simpler approach: after aggregation, query latest price per ticker_id individually (since holdings count is small, typically <50 tickers).
3. For each holding, compute:
   - `market_price = latest_close` (None if no price data)
   - `market_value = market_price * total_qty` (None if no price)
   - `unrealized_pnl = (market_price - avg_cost) * total_qty` (None if no price) — per D-08-03
   - `unrealized_pnl_pct = unrealized_pnl / total_cost * 100` (None if no price)
4. Also fetch Ticker.symbol and Ticker.name for each ticker_id
5. Return list of dicts matching HoldingResponse fields: `{symbol, name, quantity, avg_cost, market_price, market_value, total_cost, unrealized_pnl, unrealized_pnl_pct}`

**Method: `get_summary()` → dict** — per PORT-06
1. `total_invested`: Sum of `price * quantity` for all BUY trades (from trades table, not lots — includes fully consumed lots)
2. `total_market_value`: Sum of `market_price * quantity` across all open holdings (from get_holdings or computed directly)
3. `total_realized_pnl`: Sum of realized P&L across all SELL trades. Compute by replaying: for each sell trade, sum `(sell_price - lot.buy_price) * consumed_qty - sell_fees`. OR simpler: store realized_pnl alongside sell trade in record_trade and sum from trades.
   - **Recommended approach**: Add a helper `_get_total_realized_pnl()` that recomputes from lot history. For simplicity: total_realized_pnl = sum of all sell trade P&Ls. Since we don't store realized_pnl per sell in DB (only return it), recompute: `sum(sell_price * sell_qty) - sum(buy_price * consumed_qty_per_lot) - sum(sell_fees)`. This is complex.
   - **Simpler approach**: Compute from holdings. `total_realized_pnl = total sell revenue - total cost of sold shares - total sell fees`. Get this from: sum(price*quantity) for SELL trades - (sum(buy_price*quantity) for all lots MINUS sum(buy_price*remaining_quantity) for open lots) - sum(fees) for SELL trades.
   - Actually, simplest: `realized = Σ(sell_trade.price * sell_trade.quantity) - Σ(original_cost_of_consumed_portions) - Σ(sell_trade.fees)`. Where `consumed_cost = Σ(lot.buy_price * (lot.quantity - lot.remaining_quantity))` for all lots. And `sell_revenue = Σ(trade.price * trade.quantity) WHERE trade.side = 'SELL'`. So: `realized = sell_revenue - consumed_cost - sell_fees`.
4. `total_unrealized_pnl`: Sum of unrealized_pnl from holdings
5. `total_return_pct`: `(total_realized_pnl + total_unrealized_pnl) / total_invested * 100` if total_invested > 0, else None — per D-08-03
6. `holdings_count`: Number of distinct tickers with remaining_quantity > 0
7. Return dict matching PortfolioSummaryResponse fields

**Method: `get_trades(ticker_symbol=None, side=None, limit=50, offset=0)` → dict** — per PORT-07
1. Base query: `select(Trade, Ticker.symbol).join(Ticker, Trade.ticker_id == Ticker.id)`
2. Apply optional filters:
   - If `ticker_symbol`: `.where(Ticker.symbol == ticker_symbol.upper())`
   - If `side`: `.where(Trade.side == side.upper())`
3. Order by `Trade.trade_date.desc(), Trade.id.desc()`
4. Count total (before pagination): `select(func.count()).select_from(subquery)`
5. Apply `.limit(limit).offset(offset)`
6. Return `{"trades": [TradeResponse-shaped dicts], "total": count}`
   - Format dates: `trade.trade_date.isoformat()`, `trade.created_at.isoformat()`
   - `realized_pnl` field: set to None for all trades in history (realized P&L is only returned at sell time from record_trade; historical view shows trade data)

CRITICAL: Use `Decimal` for all intermediate money computation. Convert to `float` only at the return boundary (to match Pydantic schema float fields). Use `float(value)` or `round(float(value), 2)` when building response dicts.
