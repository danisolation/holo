| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, gap between chart period tabs |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section padding, space between chart cards |
| xl | 32px | Layout gaps between major sections (summary → charts → tables) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none — existing spacing patterns preserved from prior phases.

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Geist |
| Label / Caption | 12px (`text-xs`) | 400 (regular) | 1.33 | Geist |
| Heading (section) | 18px (`text-lg`) | 600 (semibold) | 1.33 | Geist |
| Display (page title) | 24px (`text-2xl`) | 600 (semibold) | 1.2 | Geist |

**Scale:** 4 sizes (12 / 14 / 18 / 24). **Weights:** 2 (400 regular / 600 semibold). No `text-[Npx]` arbitrary values.

Monospace (`font-mono`): Used for all numeric financial values (VND amounts, quantities, percentages, dates) — Geist Mono. Consistent with existing `trade-form.tsx`, `holdings-table.tsx`, `trade-history.tsx` patterns.

## Visual Hierarchy

**Primary focal point:** Performance line chart — largest visual element on the portfolio page, draws the eye via the blue area gradient. Summary cards (including new dividend card) provide key metrics above. Allocation pie chart sits beside the performance chart as a secondary visualization.

**Page layout order (top to bottom):**
1. Page header + "Thêm giao dịch" button + "Nhập CSV" button
2. Summary cards (5 cards: existing 4 + new dividend card)
3. Charts section (performance line chart + allocation pie chart side by side)
4. Holdings table
5. Trade history (with edit/delete actions per row)


**Hook:** `usePerformanceData(period: string)` — fetches `GET /api/portfolio/performance?period={period}`. staleTime: 5 minutes. queryKey: `["portfolio-performance", period]`.

### 4. TradeEditDialog (NEW)

**What:** Modal dialog for editing an existing trade. Prefilled with current trade data.
**Where:** Triggered from an "Edit" action button in the trade history table (each row).
**Component:** shadcn `Dialog` + `DialogContent` + form fields (reuses trade form field pattern from `trade-form.tsx`).

**Trigger:** `Pencil` icon button (`variant="ghost"`, `size="icon"`, `h-8 w-8`) in new Actions column of trade history table.

**Dialog spec:**

| Property | Value |
|----------|-------|
| Max width | `sm:max-w-md` (matches existing TradeForm dialog) |
| Title | `Sửa giao dịch` |
| Description | `Chỉnh sửa thông tin giao dịch. FIFO sẽ được tính lại tự động.` |

**Form fields (prefilled from trade data):**

| Field | Input Type | Label | Prefilled Value | Editable |
|-------|-----------|-------|-----------------|----------|
| Symbol | `Input` (readonly) | `Mã CK` | `trade.symbol` | No — `readOnly`, `bg-muted cursor-not-allowed` |
| Side | Button toggle | `Loại` | `trade.side` | Yes |
| Quantity | `Input type="number"` | `Số lượng` | `trade.quantity` | Yes |
| Price | `Input type="number"` | `Giá (VND)` | `trade.price` | Yes |
| Trade Date | `Input type="date"` | `Ngày GD` | `trade.trade_date` | Yes |
| Fees | `Input type="number"` | `Phí (VND)` | `trade.fees` | Yes |

**Footer buttons:**

| Button | Variant | Label | Behavior |
|--------|---------|-------|----------|
| Cancel | `outline` | `Hủy` | Closes dialog |
| Submit | `default` | `Lưu thay đổi` | Submits PUT request |

**States:**

| State | Behavior |
|-------|----------|
| Submitting | Button disabled, text `"Đang lưu..."` |
| Success | Dialog closes, toast/inline success, queries invalidated |
| Error | Inline error text `text-xs text-[#ef5350]` below form |

**Hook:** `useUpdateTrade()` — mutation wrapping `PUT /api/portfolio/trades/{id}`. On success: invalidate `["portfolio-holdings"]`, `["portfolio-summary"]`, `["portfolio-trades"]`, `["portfolio-performance"]`, `["portfolio-allocation"]`.

## Interaction Contracts

### Performance Chart Period Switching

| Step | What Happens |
|------|-------------|
| 1 | User clicks period tab (e.g., `6T`) |
| 2 | `period` state updates to `"6M"` |
| 3 | `usePerformanceData("6M")` fires — if cached, renders instantly (staleTime) |
| 4 | If not cached: chart area shows skeleton while fetching |
| 5 | Chart re-renders with new date range data |

No page navigation. Tabs are client-side state only.

### Allocation Mode Toggle

| Step | What Happens |
|------|-------------|
| 1 | User clicks mode tab (e.g., `Ngành`) |
| 2 | `mode` state updates to `"sector"` |
| 3 | `useAllocationData("sector")` fires |
| 4 | Pie chart animates transition to new slice data |
| 5 | Legend updates with sector names |

### Trade Edit Flow

| Step | What Happens |
|------|-------------|
| 1 | User clicks Pencil icon on a trade row |
| 2 | `TradeEditDialog` opens, form prefilled with trade data |
| 3 | User modifies fields (quantity, price, date, fees, side) |
| 4 | User clicks `Lưu thay đổi` |
| 5 | `PUT /api/portfolio/trades/{id}` fires |
| 6 | On success: dialog closes, all portfolio queries invalidated, data refreshes |
| 7 | On error: inline error shown, dialog stays open for retry |

### Trade Delete Flow

| Step | What Happens |
|------|-------------|
| 1 | User clicks Trash2 icon on a trade row |
| 2 | `TradeDeleteConfirm` dialog opens with trade summary |
| 3 | User clicks `Xóa giao dịch` (destructive button) |
| 4 | `DELETE /api/portfolio/trades/{id}` fires |
| 5 | On success: dialog closes, all portfolio queries invalidated |
| 6 | On error: inline error shown, dialog stays open |

### CSV Import Flow

| Step | What Happens |
|------|-------------|
| 1 | User clicks `Nhập CSV` button in page header |
| 2 | Step 1 (Upload): CSVImportDialog opens with drop zone |
| 3 | User drops or selects a .csv file |
| 4 | File name and size shown; `Tiếp tục` button enables |
| 5 | User clicks `Tiếp tục` — file uploads as `POST /api/portfolio/import?dry_run=true` |
| 6 | Step 2 (Preview): CSVPreviewTable renders with parsed rows and validation status |
| 7 | User reviews rows — errors highlighted red, warnings amber |
| 8 | If errors: `Xác nhận nhập` button disabled; user must go back and fix CSV |
| 9 | If valid: user clicks `Xác nhận nhập {N} giao dịch` |
| 10 | `POST /api/portfolio/import?dry_run=false` fires with same file/data |
| 11 | Step 3 (Result): Success message shown |
| 12 | User clicks `Đóng` — dialog closes, all portfolio queries invalidated |

**Back navigation:** User can go Step 2 → Step 1 (re-upload a different file). Step 3 is terminal (close only).

**Dialog close:** Clicking overlay or X during Step 1/2 shows no extra confirmation (no data loss — file not committed). During Step 3 (after successful import), closing is safe.

## New Hooks

```typescript
// Performance chart data
usePerformanceData(period: string)
  → GET /api/portfolio/performance?period={period}
  → queryKey: ["portfolio-performance", period]
  → staleTime: 5 * 60 * 1000

// Allocation chart data
useAllocationData(mode: "ticker" | "sector")
  → GET /api/portfolio/allocation?mode={mode}
  → queryKey: ["portfolio-allocation", mode]
  → staleTime: 2 * 60 * 1000

// Trade edit mutation
useUpdateTrade()
  → PUT /api/portfolio/trades/{id}
  → onSuccess: invalidate portfolio-holdings, portfolio-summary, portfolio-trades, portfolio-performance, portfolio-allocation

// Trade delete mutation
useDeleteTrade()
  → DELETE /api/portfolio/trades/{id}
  → onSuccess: invalidate portfolio-holdings, portfolio-summary, portfolio-trades, portfolio-performance, portfolio-allocation

// CSV dry-run upload
useCSVDryRun()
  → POST /api/portfolio/import?dry_run=true (multipart/form-data)
  → mutation, no queryKey

// CSV import commit
useCSVImport()
  → POST /api/portfolio/import?dry_run=false (multipart/form-data)
  → onSuccess: invalidate all portfolio-* query keys
```
