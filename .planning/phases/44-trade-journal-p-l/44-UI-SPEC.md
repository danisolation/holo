## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.6 | `text-sm leading-relaxed` |
| Label | 12px | 400 (regular) | 1.4 | `text-xs` |
| Heading | 24px | 700 (bold) | 1.25 | `text-2xl font-bold tracking-tight` |
| Mono | 14px | 700 (bold) | 1.4 | `font-mono text-sm font-bold` |

Weights: **2 only** — 400 (regular) for body text, labels, secondary copy + 700 (bold) for headings, mono/numeric values, emphasis.

Notes:
- All price, quantity, fee, P&L values use `font-mono text-sm font-bold` — consistent with Phase 43 pick cards
- Form field labels use `text-xs text-muted-foreground` at regular weight
- Table header cells use the existing `TableHead` component: `text-foreground font-medium` (inherits from shadcn table)
- User notes in table use `text-sm text-muted-foreground` truncated to 1 line
- Stats card values use `font-mono text-2xl font-bold` for prominent numbers

## Visual Focal Point

**Focal point: Trades table** — the sortable, color-coded trade table dominates the page. Stats cards above provide aggregate context, but the eye is drawn to the per-trade rows with their green/red P&L columns. The "Ghi lệnh" primary CTA button sits prominently in the page header to invite trade entry.

| Condition | Style |
|-----------|-------|

| SELL trade, net P&L < 0 | `text-[#ef5350]`: "-{formatVND(abs(pnl))}" with "▼" prefix |
| SELL trade, net P&L = 0 | `text-muted-foreground`: "0" |

### Form Field Specs

| Field | Type | Validation | Default | Notes |
|-------|------|-----------|---------|-------|
| Ticker | Command autocomplete (Popover + Command) | Required. Must exist in tickers table | Empty | Uses same `useTickers()` data as TickerSearch. Display: `font-mono font-bold` symbol + `text-muted-foreground` name |
| Side | 2 toggle buttons (BUY/SELL) | Required | BUY | BUY: `text-[#26a69a] bg-[#26a69a]/10 border-[#26a69a]/30`, SELL: `text-[#ef5350] bg-[#ef5350]/10 border-[#ef5350]/30`. Inactive: `variant="outline"` |
| Trade date | `Input type="date"` | Required. Cannot be future | Today | Display via native date picker. Stored as YYYY-MM-DD |
| Price | `Input type="text"` with VND formatting | Required. > 0 | Empty | Input accepts raw number, display formats with `formatVND()`. `inputMode="numeric"` |
| Quantity | `Input type="text"` | Required. > 0. Multiple of 100 | Empty | Validation error: "Số lượng phải là bội số của 100" |
| Fee override | Checkbox + `Input type="text"` | Optional. If checked, fee field required > 0 | Unchecked (auto-calc) | Default shows auto-calculated: broker_fee_pct × price × qty for each side, + 0.1% sell tax. Checkbox reveals editable input |
| Pick link | Checkbox | Optional | Auto-checked if ticker matches a daily pick from last 7 days | "Theo gợi ý AI — {symbol} #{rank} ({date})" — only shown when a matching pick exists |
| Notes | `Textarea` | Optional. Max 500 chars | Empty | `placeholder="Lý do, chiến lược, ghi chú..."` |

### Fee Auto-Calculation Display

| Line | Value | Condition |
|------|-------|-----------|
| Phí môi giới | `broker_fee_pct × price × quantity` | Always shown (both BUY and SELL) |
| Thuế bán (0.1%) | `0.001 × price × quantity` | Only shown when side = SELL |
| Tổng phí | Sum of above | Always shown |
| Format | `font-mono text-sm text-muted-foreground` | Uses `formatVND()` + " VND" suffix |

## Interaction States

### Loading State (Page)

| Element | Skeleton Spec |
|---------|---------------|
| Stats cards | 3 skeleton cards: `Skeleton className="h-20 rounded-xl"` in 3-col grid |
| Filter bar | Render immediately (not dependent on API) |
| Trades table | `Skeleton className="h-10 rounded"` repeated 5 times inside `Card` |

### Empty State (No Trades)

Displayed when `GET /api/trades` returns 0 trades.

| Element | Spec |
|---------|------|
| Container | `Card` centered within table area, `py-16 text-center` |
| Icon | `BookOpen` (lucide) `size-12 text-muted-foreground/40` |
| Heading | "Chưa có giao dịch nào" — `text-lg font-bold` |
| Body | "Ghi lệnh mua/bán đầu tiên để bắt đầu theo dõi lãi lỗ." — `text-sm text-muted-foreground mt-2 max-w-md mx-auto` |
| Action | `Button variant="default"`: "Ghi lệnh đầu tiên" — opens TradeEntryDialog |

### Error State (API Failure)

Displayed when `GET /api/trades` returns error.

| Element | Spec |
|---------|------|
| Container | `Card` centered within table area, `py-16 text-center` |
| Icon | `AlertTriangle` (lucide) `size-12 text-destructive/60` |
| Heading | "Không thể tải nhật ký" — `text-lg font-bold` |
| Body | "Đã xảy ra lỗi khi tải dữ liệu giao dịch. Vui lòng thử lại." — `text-sm text-muted-foreground mt-2` |
| Action | `Button variant="outline"`: "Thử lại" — calls `refetch()` |

### Form Validation States

| State | Display |
|-------|---------|
| Field required empty | Red border (`aria-invalid` triggers `border-destructive ring-destructive/20`), error text below: `text-xs text-destructive mt-1` |
| Quantity not multiple of 100 | Error: "Số lượng phải là bội số của 100" |
| Price ≤ 0 | Error: "Giá phải lớn hơn 0" |
| Future trade date | Error: "Ngày giao dịch không được trong tương lai" |
| SELL exceeds available lots | Error from API: "Không đủ cổ phiếu để bán. Hiện có {N} cổ {symbol}." — shown as inline error below submit button `text-sm text-destructive` |

### Trade Creation States

| State | Display |
|-------|---------|
| Submitting | Submit button shows spinner (`Loader2` icon, `animate-spin`), disabled. All form fields disabled. |
| Success | Dialog closes. Trades table + stats cards refetch (React Query invalidation). Toast not used (dialog close = implicit success). |
| API error | Inline error text in dialog footer: `text-sm text-destructive`: "Không thể lưu lệnh. Vui lòng thử lại." |

### Trade Deletion States

| State | Display |
|-------|---------|
| Confirm dialog open | Shows trade details in description for verification |
| Deleting | Destructive button shows spinner, disabled |
| Success | Dialog closes. Trades + stats refetch. |
| Error | Inline error in dialog: "Không thể xóa lệnh. Vui lòng thử lại." `text-sm text-destructive` |

### Mobile Table Strategy

On mobile, the table scrolls horizontally. The Date and Ticker columns are the first two (leftmost) so they remain visible. P&L columns scroll into view. This matches the existing `Table` component's built-in `overflow-x-auto` behavior.

## Data Flow
