# Plan 15-02: Pipeline Timeline API + Health Alerts

## Goal
Add pipeline timeline endpoint for Gantt chart visualization and proactive Telegram health alerts for sustained errors/stale data.

## Context
- Per D-15-03/D-15-04: Horizontal bar chart data from job_executions, grouped by date
- Per D-15-08: GET /health/pipeline-timeline?days=7
- Per D-15-05/D-15-06/D-15-07: health_alert_check job every 30 min, 4h cooldown, Vietnamese Telegram alerts
- JobExecution model already has started_at, completed_at, status — perfect for timeline
- MessageFormatter already has job_failure_alert method pattern to follow

## Tasks

### T-15-04: Pipeline timeline API endpoint (TDD)
Add `get_pipeline_timeline(session, days)` to HealthService:
- Query job_executions for last N days
- Group by DATE(started_at), order by started_at within each group
- Compute duration_seconds from completed_at - started_at
- Map job_id to Vietnamese job names

Response schema `PipelineTimelineResponse`:
```python
class PipelineStep(BaseModel):
    job_id: str
    job_name: str
    started_at: str
    duration_seconds: float | None
    status: str

class PipelineRun(BaseModel):
    date: str
    total_seconds: float
    steps: list[PipelineStep]

class PipelineTimelineResponse(BaseModel):
    runs: list[PipelineRun]
```

Add GET `/health/pipeline-timeline` endpoint with `days` param (default 7, max 30).

Vietnamese job name mapping (reuse _JOB_NAMES but add VN labels):

### T-15-05: Health alert service + scheduler job (TDD)
Create `HealthAlertService`:
- Module-level `_last_alert_times: dict[str, datetime]` for cooldown tracking (per D-15-06)
- `COOLDOWN_HOURS = 4`
- `check_and_alert(session)`:
  1. Check consecutive job failures (≥3 same job_id with status="failed" in last 24h)
  2. Check stale data (any source > 2× threshold from _FRESHNESS_SOURCES)
  3. Check DB pool exhaustion (checked_out / size > 0.8)
  4. For each triggered condition, check cooldown. If expired, send Telegram alert.
- Uses existing HealthService methods for data freshness
- Uses MessageFormatter.health_alert() for message formatting

Add `MessageFormatter.health_alert(alert_type, details)` class method:
- Vietnamese HTML message with severity emoji
- Lists affected items
- Includes link text "(Xem chi tiết: /dashboard/health)"

Wire `health_alert_check` scheduler job:
- Runs every 30 minutes, all day (not just market hours)
- Job ID: "health_alert_check"
- Add to _JOB_NAMES mapping
- Never-raises pattern (try/except with logging)

**Tests:** Test consecutive failure detection. Test stale data detection. Test cooldown prevents re-alerting within 4h. Test cooldown expires after 4h. Test Telegram message format. Test scheduler job is registered.
