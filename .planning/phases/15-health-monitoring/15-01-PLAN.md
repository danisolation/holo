# Plan 15-01: Gemini Usage Tracking — Model + Service + Migration

## Goal
Track Gemini API token usage per call and expose aggregated usage data through the health API so the dashboard can show consumption vs free-tier limits.

## Context
- Per D-15-01: New `gemini_usage` table: id, analysis_type, batch_size, prompt_tokens, completion_tokens, total_tokens, model_name, created_at
- Per D-15-02: Free tier limits hardcoded in config: 15 RPM, 1500 RPD, 1M tokens/day
- Per D-15-09: New GET /health/gemini-usage?days=7 endpoint
- AI service already logs `response.usage_metadata.total_token_count` — we intercept and persist

## Tasks

### T-15-01: GeminiUsage model + migration (TDD)
Create `GeminiUsage` SQLAlchemy model with columns: id (BigInteger PK), analysis_type (String 30), batch_size (Integer), prompt_tokens (Integer), completion_tokens (Integer), total_tokens (Integer), model_name (String 50), created_at (TIMESTAMP with timezone, server_default=now). Create Alembic migration 009. Register in `models/__init__.py`.

**Tests:** Verify model fields exist with correct types. Verify migration runs up/down cleanly.

### T-15-02: GeminiUsageService + recording hook (TDD)
Create `GeminiUsageService` with:
- `record_usage(session, analysis_type, batch_size, usage_metadata, model_name)` — inserts row
- `get_daily_usage(session, days=7)` — returns daily aggregates (date, total_tokens, total_requests, breakdown by analysis_type)
- `get_today_usage(session)` — returns today's totals + per-type breakdown

Modify `AIAnalysisService._call_gemini()` to call `record_usage()` after each successful Gemini response. The usage_metadata object has `prompt_token_count`, `candidates_token_count`, `total_token_count`.

**Tests:** Test record_usage inserts correctly. Test get_daily_usage aggregation. Test get_today_usage with breakdown. Test AI service calls record_usage (mock).

### T-15-03: Gemini usage API endpoint (TDD)
Add GET `/health/gemini-usage` endpoint with optional `days` query param (default 7, max 30).
Response schema `GeminiUsageResponse`:
```python
class GeminiUsageTodayBreakdown(BaseModel):
    analysis_type: str
    requests: int
    tokens: int

class GeminiUsageToday(BaseModel):
    requests: int
    tokens: int
    limit_requests: int  # 1500
    limit_tokens: int    # 1000000
    breakdown: list[GeminiUsageTodayBreakdown]

class GeminiUsageDaily(BaseModel):
    date: str
    tokens: int
    requests: int

class GeminiUsageResponse(BaseModel):
    today: GeminiUsageToday
    daily: list[GeminiUsageDaily]
```

Add free-tier limit constants to config or hardcode in schema (per D-15-02).

**Tests:** Test endpoint returns valid schema. Test with 0 usage returns zeros. Test days parameter validation.
