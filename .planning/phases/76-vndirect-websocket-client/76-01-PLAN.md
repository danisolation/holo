  <action>
Create `backend/app/services/vndirect_ws_client.py` with:

1. **Module-level parse functions** (pure, testable):
   - `parse_sp_message(raw: str) -> dict | None` — VNDirect SP messages are pipe-delimited strings like `SP|VNM|82500|85000|80000|82000|83000|84000|81000|1500000|125000000000|500|0.61`. Parse into structured dict. Fields: symbol, price (last/match), ceiling, floor, ref_price, open, high, low, volume, value, change (absolute), change_pct. Prices are in VND (integers). Return None if parsing fails.
   - `parse_ba_message(raw: str) -> dict | None` — BA messages like `BA|VNM|82000|1000|82100|2000|82200|3000|82500|1500|82400|800|82300|500`. Parse into {symbol, bids: [{price, volume}×3], asks: [{price, volume}×3]}. Bids descending, asks ascending. Return None on failure.

2. **VNDirectWSClient class**:
   - `__init__(self, symbols: list[str], on_price_update: Callable[[dict[str, dict]], Awaitable[None]], on_bid_ask_update: Callable[[dict[str, dict]], Awaitable[None]] | None = None)`
   - `async def start(self)` — Main loop: check market hours → connect → subscribe → read messages → parse → callback. On disconnect, backoff and retry. Outside market hours, sleep until next session.
   - `async def stop(self)` — Graceful shutdown (set _running=False, close WS)
   - `_should_connect(self) -> bool` — Reuse `is_market_open()` from realtime_price_service
   - `_calculate_backoff(self, attempt: int) -> float` — `min(2**attempt, 60.0)` starting at 1s
   - `_seconds_until_next_session(self) -> float` — Calculate sleep duration until next market open
   - `_subscribe(self, ws)` — Send subscription message for self._symbols after connect
   - `_handle_message(self, raw: str)` — Route to parse_sp or parse_ba, call appropriate callback

   WebSocket protocol (based on VNDirect reference implementation):
   - Connect to `wss://price-cmc-04.vndirect.com.vn/realtime/websocket`
   - Messages are newline-delimited text frames
   - Subscribe by sending ticker codes after connection
   - Use `websockets` library (already in requirements)

3. **Tests** in `backend/tests/test_vndirect_ws_client.py`:
   - Test parse_sp_message with valid SP string → correct dict
   - Test parse_sp_message with malformed input → None
   - Test parse_ba_message with valid BA string → correct dict with 3 bids, 3 asks
   - Test parse_ba_message with malformed input → None
   - Test _calculate_backoff at various attempts (0→1, 1→2, 5→32, 7→60 cap)
   - Test _should_connect respects market hours (patch _now_vn)
