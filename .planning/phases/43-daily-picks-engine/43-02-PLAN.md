1. **Pure computation functions (module-level, for easy testing):**

   `compute_composite_score(trading_signal_confidence: int, combined_score: int, safety_score: float) -> float`:
   Returns `trading_signal_confidence * 0.4 + combined_score * 0.3 + safety_score * 0.3`

   `is_affordable(capital: int, price: float) -> bool`:
   Returns `price * 100 <= capital` (100 = HOSE lot size)

   `compute_safety_score(atr_14: float, adx_14: float, avg_volume: int, current_price: float) -> float`:
   Per RESEARCH.md formulas:
   - `atr_pct = (atr_14 / current_price * 100) if current_price > 0 else 10`
   - `atr_score = max(0, min(10, (5 - atr_pct) * 2.5))`
   - `adx_score = max(0, min(10, adx_14 / 5))`
   - `vol_score = max(0, min(10, avg_volume / 50000))`
   - Returns `(atr_score + adx_score + vol_score) / 3`

   `extract_trading_plan(raw_response: dict) -> dict`:
   Extracts from `raw_response["long_analysis"]["trading_plan"]` + `raw_response["long_analysis"]["confidence"]`.
   Returns dict with keys: `entry_price`, `stop_loss`, `take_profit_1`, `take_profit_2`, `risk_reward_ratio`, `position_size_pct`, `confidence`.

   `compute_position_sizing(capital: int, entry_price: float, position_pct: int) -> dict`:
   - Cap `position_pct` at 30 max: `position_pct = min(position_pct, 30)`
   - `max_spend = capital * position_pct / 100`
   - `lot_size = 100`
   - `lots = int(max_spend / (entry_price * lot_size))`
   - `if lots < 1: lots = 1` (minimum 1 lot if affordable)
   - `shares = lots * lot_size`
   - `total_vnd = int(shares * entry_price)`
   - `capital_pct = round(total_vnd / capital * 100, 1)`
   - Returns `{"shares": shares, "total_vnd": total_vnd, "capital_pct": capital_pct}`

   `generate_rejection_reason(rsi: float | None, atr_pct: float, adx: float | None, avg_volume: int, composite_score: float) -> str`:
   Per RESEARCH.md — checks RSI > 70 ("RSI overbought"), ATR% > 4 ("Biến động cao"), ADX < 20 ("Xu hướng yếu"), volume < 100k ("Volume quá thấp"). Fallback: "Điểm tổng hợp thấp hơn ({score})". Returns max 2 reasons joined by ", ".

   `build_explanation_prompt(picks_data: list[dict]) -> str`:
   Builds a prompt string for Gemini that includes all pick symbols with their entry prices and composite scores. Use the system instruction from RESEARCH.md (`PICK_EXPLANATION_SYSTEM_INSTRUCTION`). The user prompt lists each pick with: symbol, entry_price, stop_loss, take_profit_1, composite_score.

```python
async def daily_pick_generation():
    """Generate daily stock picks after all analysis completes.
    
    Chains after daily_hnx_upcom_analysis. Scores existing trading signals,
    filters by capital/safety, generates Vietnamese explanations via Gemini.
    """
    logger.info("=== DAILY PICK GENERATION START ===")
    async with async_session() as session:
        job_svc = JobExecutionService(session)
        execution = await job_svc.start("daily_pick_generation")
        try:
            service = PickService(session)
            result = await service.generate_daily_picks()
            status = "success" if result.get("picked", 0) > 0 else "partial"
            await job_svc.finish(
                execution,
                status=status,
                result_summary=result,
            )
            logger.info(f"=== DAILY PICK GENERATION DONE: {result} ===")
        except Exception as e:
            logger.error(f"Pick generation failed: {e}")
            await job_svc.finish(execution, status="failed", result_summary={"error": str(e)})
            raise
```
