**Step 1: Add cachetools to requirements.txt**

Add `cachetools>=5.5,<6` as a new line in `backend/requirements.txt` after the existing dependencies.

**Step 2: Implement TTLCache in tickers.py**

At the top of `backend/app/api/tickers.py`, add import:
```python
from cachetools import TTLCache
```

After the `ALLOWED_ORDERS` dict (around line 144), add the cache instance:
```python
# In-memory cache for market overview — 60s TTL, max 32 entries
_market_overview_cache: TTLCache = TTLCache(maxsize=32, ttl=60)
```

In the `market_overview` function, add cache logic. The approach is inline (not decorator) because the function is async and cachetools decorators don't support async natively.

After the parameter validation (after the three HTTPException checks for exchange, sort, order), add cache lookup:

```python
    # Check cache first
    cache_key = f"{exchange}:{sort}:{order}:{top}"
    cached = _market_overview_cache.get(cache_key)
    if cached is not None:
        return cached
```

Then at the very end of the function, before `return items`, store in cache:

```python
    # Cache the result
    _market_overview_cache[cache_key] = items

    return items
```

Replace the existing `return items` (the last line of the function) with the two lines above.

This caches the fully sorted and sliced `items` list. The cache key includes all 4 query parameters so different parameter combinations get separate cache entries. TTL=60s means stale data is at most 1 minute old — acceptable for a personal stock dashboard.

- Free tier gives 50 monitors — we only need 1
