2. In `simulator_service.py`, add two methods to SimulatorService:

   a. `get_equity_history(self) -> dict`:
      - Get portfolio
      - Query ALL trades ordered by trade_date ASC
      - Reconstruct equity timeline by replaying trades:
        - Start with starting_capital on the earliest trade date
        - For each trade date, compute: cash balance after that trade + market value of remaining positions at that day's close prices
        - For market value: for each open position on that date, get DailyPrice.close × 1000
        - Group by date (one data point per day that has a trade)
        - Also include today's equity as the last point
      - Return {"history": [...points], "starting_capital": float(portfolio.starting_capital)}
      
      SIMPLIFIED APPROACH (avoid complex replay): 
      - Get all unique trade dates + today
      - For each date, calculate what the portfolio looked like AT END of that date:
        - Cash: starting_capital - sum(buy costs up to date) + sum(sell proceeds up to date)
        - Market value: sum(remaining_qty × DailyPrice.close × 1000) for each open lot at that date
      - This is expensive for many dates. BETTER approach:
        - Just compute equity at each trade date (when portfolio changes) + today
        - For trades: track running cash, use the trade's price for market value approximation
      
      SIMPLEST CORRECT APPROACH:
      - Query all trades ordered by date
      - Track running_cash starting from starting_capital
      - At each trade: adjust cash (buy reduces, sell increases by proceeds)
      - At each unique trade_date, record a point: {date, equity = running_cash + sum(open_qty × latest_price_at_date)}
      - For the "latest_price_at_date", use the trade price for that ticker (close approximation)
      - Add final point for today using current prices (reuse get_portfolio logic)
      
      ACTUALLY SIMPLEST: For equity curve, just track the cash component + approximate market value:
      - For each trade date, after all trades on that date are processed:
        - equity_point = current_cash_at_that_point + market_value_of_open_positions_at_trade_prices
      - This avoids N queries for daily prices per date
      - The final/current point uses actual latest prices (already done in get_portfolio)

   b. `get_pnl_timeline(self) -> dict`:
      - Get portfolio
      - Query ALL trades (both BUY and SELL) ordered by trade_date ASC, id ASC
      - Build list with running cumulative_pnl:
        - cumulative starts at 0
        - For SELL trades: cumulative += net_pnl
        - For BUY trades: net_pnl = None, cumulative unchanged
      - For each trade, resolve ticker_symbol
      - Return {"entries": [...], "total_realized_pnl": final_cumulative}
