## Visual Focal Point

**Focal point: Risk suggestion banner (when present) → Behavior insight cards** — When a pending risk suggestion exists, the amber warning banner at the top demands immediate attention with two action buttons. When no suggestion is pending, the existing coach page sections (picks, performance, history) remain the focal point. The behavior insights section at the bottom provides passive reflection — habit badges surface unconscious patterns, viewing stats show bias, and sector preferences reveal where the user actually performs well.

#### Risk Banner States

| State | Display |
|-------|---------|
| No pending suggestion | Banner not rendered (no DOM) |
| Pending suggestion | Full banner with both buttons enabled |
| Accepting (API call in flight) | "Đồng ý giảm" button shows `Loader2 animate-spin`, disabled. "Giữ nguyên" disabled |
| Rejecting (API call in flight) | "Giữ nguyên" button shows `Loader2 animate-spin`, disabled. "Đồng ý giảm" disabled |
| Success (accepted) | Banner removed. Toast not used (banner disappearance = implicit success). Invalidate `["profile"]` + `["picks", "today"]` (risk level change affects picks) |
| Success (rejected) | Banner removed. No further side effects |
| Error | Inline error text below buttons: `text-xs text-destructive`: "Không thể xử lý. Thử lại sau." |

## Passive Behavior Tracking (Invisible)

The `useBehaviorTracking` hook sends behavior events silently. It does NOT render any UI. Events are fire-and-forget — failures are silently ignored (no toast, no retry).

### Hook Spec

```typescript
function useBehaviorTracking(
  eventType: "ticker_view" | "search_click" | "pick_click",
  tickerSymbol?: string
): void
```

| Property | Value |
|----------|-------|
| Trigger | `useEffect` fires on mount (and when tickerSymbol changes) |
| Debounce | Max 1 event per ticker per 5 minutes (tracked in a `Set<string>` in module scope) |
| API call | `POST /api/behavior/event` with `{ event_type, ticker_symbol, metadata: {} }` |
| Error handling | `catch(() => {})` — silent swallow. No user-facing error |
| Network | Uses plain `fetch()` — NOT React Query mutation (no cache invalidation, no retries) |

### Integration Points

| Location | Event Type | Ticker Source |
|----------|-----------|--------------|
| `src/app/ticker/[symbol]/page.tsx` | `"ticker_view"` | URL param `symbol` |
| `src/components/ticker-search.tsx` (onSelect callback) | `"search_click"` | Selected ticker symbol |
| `src/components/pick-card.tsx` (card click/expand) | `"pick_click"` | Pick's `ticker_symbol` |

Each behavior card handles errors independently. Cards that fail do NOT block other cards.

| Element | Spec |
|---------|------|
| Container | `Card` with `CardContent` |
| Title | Still renders in `CardHeader` |
| Body | `text-sm text-muted-foreground py-4`: "Không thể tải dữ liệu. Thử lại sau." |
| Action | `Button variant="ghost" size="sm"`: "Thử lại" — calls `refetch()` |
| Behavior | Error in one card does NOT affect the other cards. Each has its own hook + error boundary |

## Number Formatting

| Type | Format | Example | Implementation |
|------|--------|---------|----------------|
| View count | Integer + " lượt" | "28 lượt" | `number.toString() + " lượt"` |
| Win rate % | 1 decimal place + "%" | "72.0%" | `winRate.toFixed(1) + "%"` |
| VND P&L (sector) | Sign prefix + thousands separator | "+3,250,000" / "-500,000" | `(val > 0 ? "+" : "-") + formatVND(Math.abs(val))` |
| Risk level | Integer (1-5) | "3" | Direct `number.toString()` |
| Habit count | Integer in parentheses | "(3)" | `"(" + count.toString() + ")"` |
| Trade count (insufficient note) | Integer | "2" | Direct `number.toString()` |
| Date | DD/MM/YYYY | "23/04/2026" | `formatDateVN()` from `lib/format.ts` |

## Accessibility

| Element | Requirement |
|---------|-------------|
| Risk banner | `role="alert"` + `aria-live="polite"` — announces to screen readers when it appears |
| Risk banner buttons | Clear text labels ("Đồng ý giảm" / "Giữ nguyên") — no icon-only buttons |
| Habit badges | Text label includes count — never color-only ("Bán sớm (3)") |
| Habit card icons | `aria-hidden="true"` on badge icons — text label carries meaning |
| Viewing stats list | Semantic ordered list using `ol` element with `list-none` styling |
| Sector preferences list | Semantic ordered list using `ol` element with `list-none` styling |
| Win rate / P&L values | Sign prefix (+/-) always present — never color-only for direction |
| Concentration warning | Text label ("Tập trung: {sector}") — not color-only |
| Behavior event tracking | Fully invisible — no UI impact, no focus changes, no announcements |
| Error retry buttons | Focus returned to retry button after failed refetch attempt |
