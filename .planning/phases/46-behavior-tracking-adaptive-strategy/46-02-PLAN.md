<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → POST /behavior/event | Untrusted event data crosses boundary |
| Frontend → POST /risk-suggestion/{id}/respond | Untrusted action parameter crosses boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-46-01 | Tampering | POST /behavior/event event_type | mitigate | Pydantic regex `^(ticker_view\|search_click\|pick_click)$` rejects invalid values at schema level |
| T-46-02 | Tampering | POST /risk-suggestion/{id}/respond action | mitigate | Pydantic regex `^(accept\|reject)$` rejects invalid values |
| T-46-03 | DoS | POST /behavior/event flooding | accept | Single-user app; frontend debounces 5 min/ticker; backend event_type whitelist limits scope |
| T-46-04 | Tampering | risk_suggestion status manipulation | mitigate | Service validates suggestion_id exists AND status="pending" before allowing response; returns 404 otherwise |
| T-46-05 | Elevation | risk_level modification bypass | mitigate | Risk level only changes through respond_to_risk_suggestion → UserRiskProfile update; no direct profile modification endpoint for risk_level alone |
</threat_model>
