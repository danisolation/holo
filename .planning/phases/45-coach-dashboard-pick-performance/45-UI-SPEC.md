## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.6 | `text-sm leading-relaxed` |
| Label | 12px | 400 (regular) | 1.4 | `text-xs` |
| Heading | 24px | 700 (bold) | 1.25 | `text-2xl font-bold tracking-tight` |
| Mono | 14px | 700 (bold) | 1.4 | `font-mono text-sm font-bold` |

Weights: **2 only** — 400 (regular) for body text, labels, secondary copy + 700 (bold) for headings, mono/numeric values, emphasis.

Notes:
- Performance card values use `font-mono text-2xl font-bold` for prominent numbers (same pattern as `TradeStatsCards`)
- Performance card labels use `text-xs text-muted-foreground` at regular weight
- Pick history table uses same column styles as `TradesTable` — `font-mono text-sm font-bold` for prices, `text-sm` for date
- Outcome badge text uses `text-xs font-bold` inside Badge
- Streak emoji ("🔥"/"❄️") renders inline with `text-2xl font-bold` value, no separate styling needed

## Visual Focal Point

**Focal point: Performance cards + Today's picks** — the 4-card performance grid at top gives instant feedback on coaching effectiveness, followed immediately by today's picks grid. The pick history table at the bottom provides deep-dive accountability for all past picks. Open trades section bridges the "what AI said" (picks) with "what I did" (trades).

## Number Formatting

| Type | Format | Example | Implementation |
|------|--------|---------|----------------|
| Win rate % | 1 decimal place + "%" | "68.5%" | `winRate.toFixed(1) + "%"` |
| VND P&L | Sign prefix + thousands separator + "VND" | "+4,500,000 VND" | `"+" + formatVND(val) + " VND"` |
| R:R ratio | "1:" + 1 decimal | "1:2.4" | `"1:" + ratio.toFixed(1)` |
| Streak count | Integer | "3" | Direct `number.toString()` |
| Return % | Sign prefix + 1 decimal + "%" | "+8.2%" / "-5.3%" | `(val > 0 ? "+" : "") + val.toFixed(1) + "%"` |
| Days held | Integer | "5" | Direct `number.toString()` |
| VND price (table) | Thousands separator, no decimals | "60,000" | `formatVND()` from `lib/format.ts` |
| Date | DD/MM/YYYY | "23/04/2026" | `formatDateVN()` from `lib/format.ts` |

### Mobile Table Strategy

Both Open Trades and Pick History tables scroll horizontally on mobile. For Pick History, Date and Mã columns are leftmost so they remain visible. The Outcome badge column is placed mid-table for quick scanning. This matches the existing `Table` component's `overflow-x-auto` behavior.
