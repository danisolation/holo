    **1. Add 3 new methods to PickService in pick_service.py:**

    **a) `async def compute_pick_outcomes(self) -> dict`:**
    Batch-processes all pending picked picks (status='picked' AND pick_outcome='pending' AND entry_price IS NOT NULL AND stop_loss IS NOT NULL AND take_profit_1 IS NOT NULL). For each pending pick:
    - Query DailyPrice rows WHERE ticker_id=pick.ticker_id AND date > pick.pick_date, ORDER BY date ASC
    - Build daily_closes list of (date, float(close))
    - Call compute_pick_outcome(entry_price, stop_loss, take_profit_1, take_profit_2, daily_closes)
    - If result["outcome"] != PENDING: update pick columns (pick_outcome, days_held, hit_stop_loss, hit_take_profit_1, hit_take_profit_2, actual_return_pct)
    - If result["outcome"] == PENDING and has days_held: update days_held only (so we track progress)
    - Commit once at the end (batch)
    - Skip `almost` status picks entirely (they don't have reliable entry/SL/TP)
    - Return {"processed": N, "updated": M, "still_pending": K}
    - IMPORTANT: Must be idempotent — only touches picks with pick_outcome='pending'

    **b) `async def get_performance_stats(self) -> dict`:**
    Compute aggregated performance metrics using SQL:
    - **win_rate**: `count(pick_outcome='winner') / count(pick_outcome IN ('winner','loser','expired')) * 100` — only counts resolved picks. Returns 0.0 if no resolved picks.
    - **total_pnl**: `SUM(trades.net_pnl) WHERE trades.daily_pick_id IS NOT NULL AND trades.side = 'SELL'` — realized P&L from linked trades ONLY (per CONTEXT.md locked decision). Returns 0 if no linked trades.
    - **avg_risk_reward**: `AVG(actual_return_pct / ABS((entry_price - stop_loss) / entry_price * 100))` for resolved picks where entry_price and stop_loss are not null. Protect against division by zero. Returns 0.0 if no data.
    - **current_streak**: Query picks ORDER BY pick_date DESC, rank ASC. Count consecutive same outcomes from most recent resolved pick backward. Positive = winning streak, negative = losing streak. Returns 0 if no resolved picks.
    - **total_closed, total_winners, total_losers**: Simple counts of resolved picks.
    - Return dict matching PickPerformanceResponse schema fields.

    **c) Upgrade `async def get_pick_history(self, page: int = 1, per_page: int = 20, status: str = "all") -> dict`:**
    Replace the placeholder implementation. New behavior:

    - Filter by outcome status: if status != "all", filter pick_outcome == status. Validate status against allowed values: ["all", "winner", "loser", "expired", "pending"]
