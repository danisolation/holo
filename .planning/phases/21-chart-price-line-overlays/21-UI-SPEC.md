### Accent Reserved For (explicit list)

- **`#26a69a`**: Entry price line, entry axis label background, entry legend swatch — consistent with existing candlestick upColor and LONG accent across the app
- **`#ef5350`**: Stop-loss price line, SL axis label background, SL legend swatch — consistent with existing candlestick downColor and BEARISH accent
- **`#42a5f5`**: TP1 price line, TP1 axis label background, TP1 legend swatch — blue, distinct from all existing chart colors (SMA20 is `#2196F3` but those are LineSeries, not price lines)
- **`#7e57c2`**: TP2 price line, TP2 axis label background, TP2 legend swatch — purple, distinct from TP1 and all existing chart colors (SMA200 is `#9C27B0` but lighter shade avoids confusion)

### Color Rationale

Entry and SL colors deliberately reuse the app's established LONG/BEARISH palette (`#26a69a` / `#ef5350`) for instant recognition. TP1 blue and TP2 purple are new additions chosen for maximum perceptual distance from each other and from existing MA overlay colors.

## Interaction States

### Data Present — Price Lines Visible

When `tradingPlan` prop is provided with valid prices, all 4 lines (or 3 if TP2 ≈ TP1) are rendered. Lines extend full chart width as horizontal dashed lines at the specified price levels. Each line has:
- A colored axis label on the right price scale showing the price value
- A title label ("Entry", "SL", "TP1", "TP2") displayed on the line itself

### No Data — No Price Lines

When `tradingPlan` prop is `undefined` or `null`:
- No price lines are created
- Legend row hides the price line entries (only MA legend shows)
- Chart renders exactly as before this phase (zero visual regression)

Source: CONTEXT.md locked decision — "If no trading signal or score=0: show no price lines"

### Data Changes — Ticker Navigation

When user navigates between tickers:
1. useEffect dependency on `tradingPlan` triggers cleanup
2. Previous price lines are removed via `candleSeries.removePriceLine(ref)` for each stored reference
3. New price lines are created for the new ticker's trading plan
4. Chart re-renders with updated lines

Source: CONTEXT.md locked decision — "Lines update when ticker changes (useEffect dependency)"

### TP2 Conditional Display

TP2 price line is only created when `Math.abs(take_profit_2 - take_profit_1) > 1`. This mirrors the existing `TradingPlanPanel` logic at `trading-plan-panel.tsx` line 96 which already gates TP2 display.

### Loading State

No special loading state for price lines. The chart already has a full-height skeleton (`<Skeleton className="h-[420px] rounded-xl" />`) while price data loads. Trading signal data loads independently via `useTradingSignal` hook. If the chart renders before the trading signal arrives, lines are simply not present until the signal data loads and triggers a re-render.

### Error State

No error state specific to price lines. If `useTradingSignal` fails, the `tradingPlan` prop remains `undefined`, and no lines are shown. This is silent degradation — the chart is still fully functional without price lines.

### No Animation/Transitions

Price lines appear/disappear instantly when data changes. No fade-in/fade-out. Consistent with existing chart behavior where MA overlays and Bollinger Bands appear without animation.

## Data Contract

### New Prop on CandlestickChart

```typescript
// Extend CandlestickChartProps in candlestick-chart.tsx
interface TradingPlanOverlay {
  entry_price: number;
  stop_loss: number;
  take_profit_1: number;
  take_profit_2: number;
}

interface CandlestickChartProps {
  priceData: PriceData[];
  indicatorData?: IndicatorData[];
  adjusted?: boolean;
  onAdjustedChange?: (adjusted: boolean) => void;
  tradingPlan?: TradingPlanOverlay;  // NEW — optional
}
```

### Data Flow (page.tsx → CandlestickChart)

```typescript
// In ticker/[symbol]/page.tsx, derive tradingPlan from useTradingSignal
const tradingPlan = useMemo(() => {
  if (!tradingSignal) return undefined;
  const direction = tradingSignal.recommended_direction;
  const analysis = direction === "long"
    ? tradingSignal.long_analysis
    : tradingSignal.bearish_analysis;
  if (analysis.confidence === 0) return undefined;
  return {
    entry_price: analysis.trading_plan.entry_price,
    stop_loss: analysis.trading_plan.stop_loss,
    take_profit_1: analysis.trading_plan.take_profit_1,
    take_profit_2: analysis.trading_plan.take_profit_2,
  };
}, [tradingSignal]);

// Pass to CandlestickChart
<CandlestickChart
  priceData={priceData}
  indicatorData={indicatorData ?? undefined}
  adjusted={adjusted}
  onAdjustedChange={setAdjusted}
  tradingPlan={tradingPlan}
/>
```

## Legend Extension

### Extended Legend Row

The existing MA legend row (line 281-294 of `candlestick-chart.tsx`) is extended with price line entries when `tradingPlan` is provided:

```
● SMA 20  ● SMA 50  ● SMA 200  ┄ BB  |  ╍ Entry  ╍ SL  ╍ TP1  ╍ TP2
```

Each price line legend entry uses the same pattern as existing MA entries:
```tsx
<span className="flex items-center gap-1">
  <span className="inline-block w-3 h-0.5" style={{ borderTop: "1px dashed #26a69a" }} /> Entry
</span>
```

The dashed style (`borderTop: "1px dashed {color}"`) distinguishes price line legends from MA solid-line legends, matching the actual line rendering on the chart.

Legend entries for price lines are conditionally rendered — only shown when `tradingPlan` prop is present.
