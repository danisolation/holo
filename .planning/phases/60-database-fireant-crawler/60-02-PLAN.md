async def _get_watchlist_ticker_map(session) -> dict[str, int]:
    # JOINs UserWatchlist.symbol → Ticker.symbol → returns {symbol: Ticker.id}
    from app.models.user_watchlist import UserWatchlist
    from app.models.ticker import Ticker
    stmt = (

Create `backend/app/crawlers/fireant_crawler.py` following the CafeFCrawler pattern exactly:

```python
"""Fireant.vn community post crawler for rumor intelligence.

Fetches posts from Fireant REST API for watchlist tickers.
Stores in rumors table with deduplication via ON CONFLICT DO NOTHING.

Key decisions (from 60-CONTEXT.md):
- Fireant REST API at restv2.fireant.vn/posts with guest JWT
- Watchlist-gated crawling only (not all 400 tickers)
- html.unescape() for Vietnamese content encoding
- 1.5s delay between ticker requests
- 30-day retention — delete older posts on each crawl
"""
```

Class structure:
- `API_URL = "https://restv2.fireant.vn/posts"`
- `MIN_CONTENT_LENGTH = 20` — filter noise posts (per STACK.md pre-filter)

Constructor `__init__(self, session: AsyncSession, delay: float | None = None)`:
- `self.session = session`
- `self.delay = delay if delay is not None else settings.fireant_delay_seconds`
- `self.post_limit = settings.fireant_post_limit`
- `self.retention_days = settings.fireant_retention_days`
- `self.headers = {"Authorization": f"Bearer {settings.fireant_token}", "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"}`

Method `async def crawl_watchlist_tickers(self) -> RumorCrawlResult`:
- Import `_get_watchlist_ticker_map` from `app.scheduler.jobs` — reuse exact same function
- Actually, to avoid circular imports, duplicate the watchlist query inline (same 5-line query pattern from jobs.py lines 70-78)
- Query UserWatchlist joined with Ticker to get `{symbol: ticker_id}` dict
- If empty, log warning and return early with zeros
- Loop through tickers with httpx.AsyncClient, call `_fetch_posts` → `_store_posts` per ticker
- Add `asyncio.sleep(self.delay)` between tickers (except last)
- Call `_cleanup_old_posts()` after all tickers done
- Commit session
- Return `RumorCrawlResult` dict with success/failed/total_posts/failed_symbols

Method `_fetch_posts_raw(self, client: httpx.AsyncClient, symbol: str) -> list[dict]`:
- Decorated with `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=2, min=2, max=8), retry=retry_if_exception(_is_retryable), reraise=True)`
- `params = {"symbol": symbol.upper(), "offset": 0, "limit": self.post_limit}`
- `resp = await client.get(self.API_URL, params=params)`
- `resp.raise_for_status()`
- `return self._parse_posts(resp.json())`

Method `_fetch_posts(self, client, symbol) -> list[dict]`:
- `return await fireant_breaker.call(self._fetch_posts_raw, client, symbol)`

Method `_parse_posts(self, posts_json: list[dict]) -> list[dict]`:
- For each post in posts_json:
  - `content = html.unescape(post.get("content", "")).strip()`
  - Skip if `len(content) < self.MIN_CONTENT_LENGTH`
  - Extract: `post_id=post["postID"]`, `content=content`, `author_name=post.get("user", {}).get("name", "Unknown")`, `is_authentic=post.get("user", {}).get("isAuthentic", False)`, `total_likes=post.get("totalLikes", 0)`, `total_replies=post.get("totalReplies", 0)`, `fireant_sentiment=post.get("sentiment", 0)`, `posted_at=datetime.fromisoformat(post["date"])`
- Return list of parsed dicts

Method `async def _store_posts(self, ticker_id: int, posts: list[dict]) -> int`:
- If not posts: return 0
- For each post: `stmt = insert(Rumor).values(ticker_id=ticker_id, **post).on_conflict_do_nothing(constraint="uq_rumors_post_id")`
- Execute and sum rowcount
- Return stored count

Method `async def _cleanup_old_posts(self)`:
- `cutoff = datetime.now(timezone.utc) - timedelta(days=self.retention_days)`
- `stmt = delete(Rumor).where(Rumor.posted_at < cutoff)`
- `result = await self.session.execute(stmt)`
- Log deleted count if > 0

Copy `_is_retryable` function from cafef_crawler.py (identical logic).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-60-04 | Tampering | Fireant JSON response | mitigate | _parse_posts validates field types, html.unescape cleans content, SQLAlchemy mapped_column enforces schema types, String(200) truncates author_name |
| T-60-05 | Denial of Service | Fireant API rate limiting | mitigate | asyncio.sleep(1.5s) between requests, circuit breaker opens after 3 failures, tenacity retry with exponential backoff |
| T-60-06 | Information Disclosure | Fireant JWT token in logs | mitigate | Token in env var only, not logged — loguru structured logging doesn't dump headers |
| T-60-07 | Elevation of Privilege | Malicious post content (XSS) | accept | Content stored as text, rendered by Next.js which auto-escapes JSX. No raw HTML rendering. |
