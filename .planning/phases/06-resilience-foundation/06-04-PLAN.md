    Refactor ALL job functions in `backend/app/scheduler/jobs.py` to follow this pattern. The key behavioral changes per CONTEXT.md decisions:

    1. **Job tracking (D-13/D-14):** Each job creates a JobExecution record at start, updates on completion
    2. **Retry failed tickers (D-06):** After first pass, collect failed_symbols and re-run only those
    3. **DLQ (D-08/D-09):** After retry, permanently failed tickers go to dead-letter queue
    4. **Graceful degradation (Pitfall 2):** Jobs return normally on partial failure (chain continues). Only raise on COMPLETE failure (0 successes).
    5. **Critical notification (D-10):** Complete failures raise exception → EVENT_JOB_ERROR fires → _on_job_error sends Telegram alert. No need to call Telegram directly from jobs.
    6. **Retry INSIDE jobs (D-07):** Re-batch is inside the job function, NOT wrapping it.

    **Add imports at top of jobs.py:**
    ```python
    from app.services.job_execution_service import JobExecutionService
    from app.services.dead_letter_service import DeadLetterService
    from app.resilience import CircuitOpenError
    ```

    **Template for data-crawling jobs** (daily_price_crawl, weekly_ticker_refresh, weekly_financial_crawl):

    ```python
    async def daily_price_crawl():
        """Daily OHLCV crawl for all active tickers.

        Runs Mon-Fri at 15:30 Asia/Ho_Chi_Minh.
        Tracks execution in job_executions table (D-13).
        Retries failed tickers once (D-06). DLQs permanent failures (D-08).
        Returns normally on partial failure (Pitfall 2). Raises on complete failure.
        """
        logger.info("=== DAILY PRICE CRAWL START ===")
        async with async_session() as session:
            job_svc = JobExecutionService(session)
            execution = await job_svc.start("daily_price_crawl")
            try:
                crawler = VnstockCrawler()
                service = PriceService(session, crawler)
                result = await service.crawl_daily()

                # D-06: Retry failed tickers only (one re-batch attempt)
                failed_symbols = result.get("failed_symbols", [])
                retried_count = 0
                if failed_symbols:
                    logger.info(f"Retrying {len(failed_symbols)} failed tickers...")
                    retry_result = await service.crawl_specific(failed_symbols) if hasattr(service, 'crawl_specific') else {"success": 0, "failed": len(failed_symbols), "failed_symbols": failed_symbols}
                    # If crawl_specific doesn't exist, try _crawl_batch with ticker_map
                    # Read PriceService to find the right method for retrying specific symbols
                    retried_count = len(failed_symbols)
                    # Merge: use retry result for final failed count
                    result["failed_symbols"] = retry_result.get("failed_symbols", failed_symbols)
                    result["failed"] = retry_result.get("failed", len(failed_symbols))
                    result["success"] = result.get("success", 0) + retry_result.get("success", 0)

                # D-08/D-09: DLQ permanently failed tickers
                final_failed = result.get("failed_symbols", [])
                if final_failed:
                    dlq_svc = DeadLetterService(session)
                    for symbol in final_failed:
                        await dlq_svc.add("daily_price_crawl", symbol, f"Persistent failure after retry")

                # Build result summary (D-13 JSONB schema)
                summary = {
                    "tickers_processed": result.get("success", 0),
                    "tickers_failed": result.get("failed", 0),
                    "tickers_skipped": result.get("skipped", 0),
                    "failed_symbols": final_failed,
                    "retried_count": retried_count,
                    "dlq_count": len(final_failed),
                }

                # Determine status
                if result.get("success", 0) == 0 and result.get("failed", 0) > 0:
                    status = "failed"
                elif result.get("failed", 0) > 0:
                    status = "partial"
                else:
                    status = "success"

                await job_svc.complete(execution, status=status, result_summary=summary)
                await session.commit()
                logger.info(f"=== DAILY PRICE CRAWL COMPLETE: {summary} ===")

                # Pitfall 2: Only raise on COMPLETE failure (breaks chain)
                if status == "failed":
                    raise RuntimeError(f"Complete crawl failure: all tickers failed")

            except Exception as e:
                if execution.status == "running":
                    await job_svc.fail(execution, error=str(e))
                    await session.commit()
                logger.error(f"=== DAILY PRICE CRAWL FAILED: {e} ===")
                raise
    ```

    The key is: DO NOT re-crawl all 400 tickers. Only the failed ones.
