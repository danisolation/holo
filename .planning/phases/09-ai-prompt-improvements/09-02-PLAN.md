**Modify all 4 `_analyze_*_batch` methods** to add a low-temperature retry at 0.05 before the existing JSON parse fallback (per D-09-07 / AI-12).

The pattern for each method (technical, fundamental, sentiment, combined):

```python
async def _analyze_technical_batch(self, ticker_data):
    prompt = self._build_technical_prompt(ticker_data)
    temp = ANALYSIS_TEMPERATURES[AnalysisType.TECHNICAL]
    sys_instr = TECHNICAL_SYSTEM_INSTRUCTION

    response = await self._call_gemini(prompt, TechnicalBatchResponse, temp, sys_instr)
    logger.debug(f"Gemini technical tokens: {response.usage_metadata.total_token_count}")
    result = response.parsed

    if result is None and response.text:
        # Phase 1: Low-temperature retry (AI-12, D-09-07)
        logger.warning("response.parsed is None, retrying at temperature=0.05")
        response = await self._call_gemini(prompt, TechnicalBatchResponse, 0.05, sys_instr)
        result = response.parsed

    if result is None and response.text:
        # Phase 2: Manual JSON parse fallback (existing behavior)
        logger.warning("Low-temp retry also failed, falling back to manual JSON parse")
        try:
            data = json.loads(response.text)
            result = TechnicalBatchResponse.model_validate(data)
        except Exception as e:
            logger.error(f"Manual parse also failed: {e}")
            logger.debug(f"Raw response text: {response.text[:500]}")

    return result
```

**Key changes from current code:**
1. The first `if result is None` block is NEW — it retries at temp=0.05 with the same prompt, schema, and system_instruction
2. The second `if result is None` block is the EXISTING JSON parse fallback, but now only triggers if the low-temp retry also failed
3. Both `if` blocks check `response.text` — if there's no text at all, neither fires
4. Log messages differentiate the two phases: "retrying at temperature=0.05" vs "Low-temp retry also failed, falling back to manual JSON parse"

Apply this **exact same pattern** to all 4 methods:
- `_analyze_technical_batch` — uses TECHNICAL_SYSTEM_INSTRUCTION, TechnicalBatchResponse
- `_analyze_fundamental_batch` — uses FUNDAMENTAL_SYSTEM_INSTRUCTION, FundamentalBatchResponse
- `_analyze_sentiment_batch` — uses SENTIMENT_SYSTEM_INSTRUCTION, SentimentBatchResponse
- `_analyze_combined_batch` — uses COMBINED_SYSTEM_INSTRUCTION, CombinedBatchResponse

**Important:** The retry is at the batch level (one retry per batch), NOT per-ticker. This is by design — per Pitfall 2 in RESEARCH.md, we limit to exactly 1 retry to avoid doubling API cost. The manual JSON parse fallback is free (no API call).
