## Component Inventory

### 1. ExchangeFilter (NEW)

**What:** Segmented tab bar for filtering by exchange.
**Where:** Market overview page (`page.tsx`), Watchlist page (`watchlist/page.tsx`), Dashboard page (`dashboard/page.tsx`).
**Component:** shadcn `Tabs` + `TabsList` + `TabsTrigger` (default variant — filled/segmented).
**Placement:** Below page title, above market stats cards. Full width on mobile, `w-fit` on desktop.

| Tab | Value | Label |
|-----|-------|-------|
| All | `"all"` | `Tất cả` |
| HOSE | `"HOSE"` | `HOSE` |
| HNX | `"HNX"` | `HNX` |
| UPCOM | `"UPCOM"` | `UPCOM` |

**Default value:** `"all"` (CONTEXT.md decision).
**State management:** Store selected exchange in zustand `useExchangeFilterStore` with `persist` middleware (key: `holo-exchange-filter`). Shared across all pages so switching on market overview persists to watchlist.
**Keyboard:** Arrow keys navigate between tabs (built into shadcn Tabs).

**Visual spec:**
- Use `TabsList` default variant (filled background `bg-muted`), height `h-9`
- Each `TabsTrigger` shows exchange name in uppercase, font-size 13px (`text-[13px]`), font-weight 500
- Active trigger gets `data-active` styling from shadcn (white bg in light, input/30 in dark)
- Optional: Show ticker count next to each tab label as `(N)` in `text-muted-foreground text-xs` — e.g., `HOSE (400)`
- Tab order: Tất cả | HOSE | HNX | UPCOM

**Mobile:** Tabs scroll horizontally if needed. Use `overflow-x-auto` on wrapper if total width exceeds viewport.

### 2. ExchangeBadge (NEW)

**What:** Color-coded badge showing which exchange a ticker belongs to.
**Where:** Watchlist table (new column), Stock list tables, Ticker search results, Ticker detail page header.
**Component:** shadcn `Badge` variant `"outline"` with exchange-specific color overrides.

**Visual spec per exchange:**

| Exchange | className |
|----------|-----------|
| HOSE | `border-[var(--exchange-hose)] text-[var(--exchange-hose-fg)] bg-[var(--exchange-hose)]/10` |
| HNX | `border-[var(--exchange-hnx)] text-[var(--exchange-hnx-fg)] bg-[var(--exchange-hnx)]/10` |
| UPCOM | `border-[var(--exchange-upcom)] text-[var(--exchange-upcom-fg)] bg-[var(--exchange-upcom)]/10` |

**Size:** `h-5` (default Badge height), `text-[11px]`, `font-semibold`, `uppercase`.
**Content:** Exchange name string — `"HOSE"`, `"HNX"`, `"UPCOM"`.
**Sorting:** When used in `@tanstack/react-table`, the exchange column is sortable (alphabetic) and filterable (by tab selection).

### 4. AnalyzeNowButton (NEW)

**What:** Button to trigger on-demand AI analysis for HNX/UPCOM tickers not in the daily analysis schedule.
**Where:** Ticker detail page (`/ticker/[symbol]/page.tsx`), below the analysis cards section header.
**When visible:** Only show when:
  1. The ticker's exchange is HNX or UPCOM, AND
  2. The ticker is NOT in the user's watchlist (watchlisted tickers get daily analysis), AND
  3. There is no recent analysis (analysis_date is older than 24h or null)

**Component:** shadcn `Button` variant `"default"` (primary), size `"sm"`.
**Icon:** `Sparkles` from lucide-react (indicates AI action), placed before text.
**Label:** `"Phân tích ngay"` (Analyze now).

**States:**

| State | Appearance | Behavior |
|-------|------------|----------|
| Idle | Primary button with Sparkles icon + "Phân tích ngay" | Clickable |
| Loading | Button disabled, icon replaced with `Loader2` spinning, text "Đang phân tích..." | Non-interactive, aria-busy="true" |
| Success | Button becomes `variant="outline"` with `Check` icon, text "Đã phân tích" | Disabled for 5s, then re-enables. Analysis cards below refresh via `queryClient.invalidateQueries` |
| Error | Destructive text below button: "Không thể phân tích. Thử lại sau." | Button re-enables for retry |
| Cooldown | Button disabled, text "Thử lại sau {N}s" | 60-second cooldown between requests to respect Gemini rate limits |

**Layout:** Inline with section heading "Phân tích AI đa chiều" — placed to the right with `ml-auto`.

### 6. API Type Updates (MODIFY `api.ts`)

## Interaction Contracts

### Exchange Tab Switching

| Step | What Happens |
|------|-------------|
| 1 | User clicks/taps a tab trigger (e.g., `HNX`) |
| 2 | `useExchangeFilterStore.setExchange("HNX")` fires |
| 3 | All consuming components re-render with filtered data |
| 4 | Market overview refetches with `?exchange=HNX` query param |
| 5 | Heatmap shows only HNX tickers |
| 6 | Stats cards (Tổng mã / Tăng giá / Giảm giá / Đứng giá) recalculate for HNX subset |
| 7 | Skeleton loading appears during refetch (staleTime handles instant switches for cached exchanges) |

**No page navigation occurs.** Tab switching is client-side state only, updating query params is optional (URL sync is deferred to future).

### On-Demand Analysis Flow

| Step | What Happens |
|------|-------------|
| 1 | User navigates to HNX/UPCOM ticker detail page |
| 2 | Analysis section shows empty state: `"Chưa có phân tích AI. Nhấn "Phân tích ngay" để bắt đầu."` |
| 3 | User clicks `"Phân tích ngay"` button |
| 4 | Button enters loading state (disabled, spinner, text changes) |
| 5 | POST `/api/analysis/{symbol}/trigger` fires |
| 6 | On success: button shows success state, analysis cards refresh and populate |
| 7 | On error: error message appears below button, button re-enables |
| 8 | 60-second cooldown timer starts regardless of success/error |

### Heatmap Filtering

| Filter | Behavior |
|--------|----------|
| `"all"` | Show all tickers from all exchanges, grouped by sector. Exchange border colors provide visual distinction. |
| `"HOSE"` | Show only HOSE tickers. Borders are all `--exchange-hose` blue. |
| `"HNX"` | Show only HNX tickers. Borders are all `--exchange-hnx` green. |
| `"UPCOM"` | Show only UPCOM tickers. Borders are all `--exchange-upcom` orange. |
