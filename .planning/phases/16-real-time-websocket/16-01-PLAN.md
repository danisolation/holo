# Plan 16-01: Backend WebSocket — Connection Manager + VCI Polling + WS Endpoint

## Goal
Create the backend WebSocket infrastructure: connection manager, VCI polling service, market hours logic, and WebSocket endpoint.

## Context
- Per D-16-01: Backend polls VCI, stores latest prices in memory, broadcasts to WebSocket clients
- Per D-16-02: `ws://host/ws/prices` — single endpoint, subscribe by symbols, 15s heartbeat
- Per D-16-03: APScheduler interval job every 30s, polls vnstock price_board()
- Per D-16-04: Market hours 9:00-11:30, 13:00-14:45 Mon-Fri VN timezone
- Per D-16-07: In-memory ConnectionManager with connect/disconnect/subscribe/broadcast
- Per D-16-09: Poll only subscribed symbols, max ~50, skip if no clients

## Tasks

### T-16-01: Market hours utility + VCI price polling (TDD)
Add to `vnstock_crawler.py`:
```python
async def fetch_price_board(self, symbols: list[str]) -> dict[str, dict]:
    """Fetch current prices for multiple symbols via VCI price_board."""
```
Uses `vnstock.explorer.vci.quote.Quote` or `Trading` module to get current prices. Returns dict keyed by symbol with price, change, change_pct, volume.

Create `realtime_price_service.py` with:
- `is_market_open() -> bool` — checks VN timezone weekday + time windows
- `get_market_session() -> str` — returns "morning", "afternoon", or "closed"
- `RealtimePriceService` class:
  - `_latest_prices: dict[str, dict]` — in-memory cache
  - `poll_and_broadcast()` — called by scheduler, fetches VCI, updates cache, calls ConnectionManager.broadcast()
  - `get_latest_prices(symbols) -> dict` — returns cached prices for symbols

Add settings to `config.py`:
- `realtime_poll_interval: int = 30` (seconds)
- `realtime_max_symbols: int = 50`

**Tests:** Test is_market_open for various times. Test get_market_session. Test poll_and_broadcast with mocked VCI. Test price cache update.

### T-16-02: WebSocket connection manager + endpoint (TDD)
Create `ws/prices.py` with:
- `ConnectionManager`:
  - `_connections: dict[WebSocket, set[str]]` — websocket → subscribed symbols
  - `connect(ws)`, `disconnect(ws)`, `subscribe(ws, symbols)`, `get_all_subscribed_symbols() -> set[str]`
  - `broadcast(prices: dict)` — sends to each client only their subscribed symbols
  - `send_heartbeat()` — sends `{"type": "heartbeat"}` to all
  - `send_market_status(is_open, session)` — sends `{"type": "market_status", ...}` to all

- WebSocket route at `/ws/prices`:
  - Accept connection, add to manager
  - Listen for subscribe messages: `{"type": "subscribe", "symbols": ["VNM", "FPT", ...]}`
  - On disconnect: remove from manager
  - Validation: max 50 symbols per subscribe, symbols must be uppercase

Wire into `main.py` — add WebSocket route to FastAPI app.

Wire scheduler jobs in `manager.py`:
- `realtime_price_poll` — interval 30s, runs always but polls VCI only when market open
- `realtime_heartbeat` — interval 15s, sends heartbeat to all clients

Add job IDs to _JOB_NAMES.

**Tests:** Test ConnectionManager subscribe/unsubscribe. Test broadcast filters by subscription. Test WebSocket endpoint accepts and handles subscribe. Test heartbeat. Test market status broadcast. Test scheduler job registration.
