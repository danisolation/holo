```python
class BacktestAnalysisService(AIAnalysisService):
    """Date-aware analysis service for backtesting.
    
    Overrides context-gathering to use as_of_date instead of latest data,
    and storage to write to backtest_analyses instead of ai_analyses.
    """
    def __init__(self, session: AsyncSession, run_id: int, as_of_date: date, api_key: str | None = None):
        super().__init__(session, api_key)
        self.run_id = run_id
        self.as_of_date = as_of_date
```

**Module-level utility function:**
```python
def apply_slippage(price: Decimal, slippage_pct: Decimal, is_buy: bool) -> Decimal:
    """Apply slippage: buying costs more, selling receives less."""
    if is_buy:
        return price * (1 + slippage_pct / 100)
    else:
        return price * (1 - slippage_pct / 100)
```

**Class: BacktestEngine**

```python
class BacktestEngine:
    """Backtest engine: replays historical sessions with Gemini AI analysis.
    
    Date-first iteration: each session processes ALL tickers → evaluate positions → next day.
    Checkpoint after each completed day for resume capability.
    """
```

**Constructor:** Takes no parameters. Uses `async_session()` for DB access internally.

**Public method: `async def run(self, run_id: int) -> None`**

This is the main entry point called by the API's BackgroundTasks. Complete flow:

1. **Load run** — query BacktestRun by id, validate status is RUNNING
2. **Get trading dates** — `SELECT DISTINCT date FROM daily_prices WHERE date BETWEEN run.start_date AND run.end_date ORDER BY date ASC`. This gives actual trading days only.
3. **Get all active tickers** — `SELECT symbol, id FROM tickers WHERE is_active = true` → dict[str, int]
4. **Resume logic (BT-05):**
   - If `run.last_completed_date` is not None, filter trading_dates to only those > last_completed_date
   - Load open positions from DB: `SELECT * FROM backtest_trades WHERE run_id = run_id AND status IN ('pending', 'active', 'partial_tp')`
   - Compute current cash: if last equity snapshot exists (`backtest_equity WHERE run_id ORDER BY date DESC LIMIT 1`), use its `cash` value. Else use `run.initial_capital`.
5. **If no resume**, set cash = run.initial_capital (Decimal)

6. **Main date loop** — `for session_date in trading_dates:`

   a. **Cancel check** — query `backtest_runs.is_cancelled WHERE id = run_id`. If True:
      - Set run.status = BacktestStatus.CANCELLED, commit, return

   b. **Run AI pipeline** — Create fresh `BacktestAnalysisService(session, run_id, session_date)` for this date.
      - Call `run_technical_analysis(ticker_filter=ticker_map)` — batches 25 tickers, 4s delay
      - Call `run_combined_analysis(ticker_filter=ticker_map)` — uses backtest technical results
      - Call `run_trading_signal_analysis(ticker_filter=ticker_map)` — generates signals
      - Wrap each in try/except: on persistent failure, log error and continue to next day (per CONTEXT.md decision)
      - NOTE: Do NOT call `run_fundamental_analysis` or `run_sentiment_analysis` — per RESEARCH.md, fundamental is quarterly (same data), sentiment historical data unavailable (combined analysis gracefully defaults to neutral/5)
      - NOTE: Do NOT acquire the module-level `_gemini_lock` here — the analysis service methods internally handle batch orchestration. The backtest runs via BackgroundTasks which shares the same event loop, and the singleton pattern ensures no concurrent backtest.

   c. **Activate PENDING trades (D+1 entry)** — per BT-03:
      - Query `backtest_trades WHERE run_id = run_id AND status = 'pending' AND signal_date < session_date`
      - For each pending trade, get today's OHLCV bar for that ticker from `daily_prices WHERE ticker_id AND date = session_date`
      - If bar exists: set status=ACTIVE, entry_price=apply_slippage(bar.open, slippage_pct, is_buy=(direction=='long')), entry_date=session_date
      - Deduct cash: `cash -= entry_price_with_slippage × quantity`
      - If bar doesn't exist (ticker not traded that day): skip, try next day

   d. **Evaluate open positions** — per BT-04:
      - Query `backtest_trades WHERE run_id = run_id AND status IN ('active', 'partial_tp')`
      - Get OHLCV bars for all ticker_ids on session_date (single query: `SELECT * FROM daily_prices WHERE ticker_id IN (...) AND date = session_date`)
      - For each position:
        * Get bar from price_map. If no bar, skip.
        * Convert bar prices to Decimal: bar_open, bar_high, bar_low
        * Call `evaluate_long_position()` or `evaluate_bearish_position()` from paper_trade_service (import directly)
        * If result is PARTIAL_TP: call `apply_partial_tp(trade, slipped_tp1)` where slipped_tp1 = apply_slippage(exit_price, slippage_pct, is_buy=False). Cash += slipped_tp1 × closed_quantity.
        * If result is CLOSED_* (SL, TP2): Apply slippage to exit price. Set trade.exit_price, trade.closed_date=session_date, trade.status. Call `calculate_pnl()` → set realized_pnl, realized_pnl_pct. Cash += slipped_exit × remaining_quantity.
        * If no status change: check timeout. Count trading days since entry_date using `SELECT COUNT(*) FROM daily_prices WHERE ticker_id = trade.ticker_id AND date BETWEEN trade.entry_date AND session_date`. Compare to TIMEOUT_TRADING_DAYS[trade.timeframe]. If exceeded: close at today's close with slippage, status=CLOSED_TIMEOUT, compute P&L, add cash back.
      - Commit after all position evaluations

   e. **Process new signals (create PENDING trades)** — per BT-03:
      - Query today's valid trading signals: `backtest_analyses WHERE run_id AND analysis_type='trading_signal' AND analysis_date=session_date AND score > 0`
      - For each signal:
        * Parse raw_response with `TickerTradingSignal.model_validate(analysis.raw_response)` (import from app.schemas.analysis)
        * Get recommended direction and corresponding directional analysis
        * Extract trading plan (entry_price, stop_loss, take_profit_1, take_profit_2, position_size_pct, timeframe, confidence, risk_reward_ratio)
        * Call `calculate_position_size(capital=cash, allocation_pct=plan.position_size_pct, entry_price=Decimal(str(plan.entry_price)))` — per SIM-02, uses CURRENT cash not initial
        * If quantity == 0: skip (not enough capital)
        * Create BacktestTrade with status=PENDING, signal_date=session_date, all price/meta fields from signal
        * session.add(trade)
      - Commit after all signals processed

   f. **Record equity snapshot** — per SIM-04:
      - positions_value = Decimal("0")
      - For each open position (query backtest_trades WHERE run_id AND status IN ('pending','active','partial_tp')):
        * Get close price for ticker on session_date from daily_prices
        * For active/partial_tp: positions_value += close_price × (quantity - closed_quantity)
        * For pending: positions_value += 0 (not yet entered, no market exposure)
      - total_equity = cash + positions_value
      - cumulative_return_pct = float((total_equity - run.initial_capital) / run.initial_capital * 100)
      - daily_return_pct: if previous equity snapshot exists, compute from previous total_equity
      - Create BacktestEquity record, session.add()

   g. **Checkpoint** — per BT-05:
      - Update `run.last_completed_date = session_date`
      - Update `run.completed_sessions += 1`
      - Commit the session (this commits equity + trades + run update atomically)
      - Log progress: `f"Backtest {run_id}: completed {session_date} ({run.completed_sessions}/{run.total_sessions})"`

7. **After loop completes:**
   - Set run.status = BacktestStatus.COMPLETED
   - Commit
   - Log completion

8. **Exception handler (wrap entire run in try/except):**
   - On any unhandled exception: set run.status = BacktestStatus.FAILED, commit, re-raise (for logging)
