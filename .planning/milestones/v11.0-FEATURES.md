# v11.0 Features — UX & Reliability Overhaul

## 1. API Performance Optimization

**Problem:** Market overview API took ~3 minutes due to unfiltered `ROW_NUMBER()` scan over 200K+ rows.

**Solution:**
- Date-bounded filter: `WHERE date >= CURRENT_DATE - 7` reduces scan to ~2,800 rows
- Composite index: `ix_daily_prices_ticker_date(ticker_id, date DESC)` via Alembic migration 027
- TTL cache: `cachetools.TTLCache(maxsize=32, ttl=60)` on market overview endpoint

**Result:** Response time ~3s (from ~3 min). Cache hits return in ~2s.

**Files:** `backend/app/api/tickers.py`, `backend/alembic/versions/027_*.py`

---

## 2. Keep-Alive Service

**Problem:** Render free tier spins down after 15 min inactivity → cold start delay.

**Solution:** UptimeRobot/cron-job.org pings `GET /` every 5-14 minutes.

**Setup:** See `KEEPALIVE-SETUP.md` (archived in milestones).

## 3. Ticker Search Fix

**Problem:** Search only showed first 50 tickers (`.slice(0, 50)` before cmdk filter). Backend capped at 100.

**Solution:**
- Removed `.slice(0, 50)` from `ticker-search.tsx` and `trade-entry-dialog.tsx`
- Changed `useTickers(undefined, undefined, 500)` to fetch all ~400
- Backend `limit` param max raised to 500

**Files:** `frontend/src/components/ticker-search.tsx`, `frontend/src/components/trade-entry-dialog.tsx`

## 5. Morning AI Refresh Chain

**Problem:** AI analysis only ran after market close (15:30). Data stale by next morning.

**Solution:**
- CronTrigger at 08:30 Mon-Fri (Asia/Ho_Chi_Minh)
- Shortened chain: price → indicators → AI → signals (skips discovery/news/sentiment)
- Scoped to watchlist tickers only (~10-30), fits Gemini 15 RPM free tier

**Files:** `backend/app/scheduler/manager.py`, `backend/app/scheduler/jobs.py`

---

## 6. AI Freshness Indicator

**Feature:** Watchlist table shows "AI" column with relative time (e.g., "2h ago") and color coding:
