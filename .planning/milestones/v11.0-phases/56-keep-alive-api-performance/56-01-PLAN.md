In `backend/app/api/tickers.py`, modify the `market_overview` function's ranked subquery (lines 176-188) to add a WHERE clause that bounds the date range.

Current code (the ranked subquery, line 176-188):
```python
ranked = (
    select(
        DailyPrice.ticker_id,
        DailyPrice.close,
        func.row_number()
        .over(
            partition_by=DailyPrice.ticker_id,
            order_by=DailyPrice.date.desc(),
        )
        .label("rn"),
    )
    .subquery("ranked")
)
```

Change to add `.where()` BEFORE `.subquery()`:
```python
ranked = (
    select(
        DailyPrice.ticker_id,
        DailyPrice.close,
        func.row_number()
        .over(
            partition_by=DailyPrice.ticker_id,
            order_by=DailyPrice.date.desc(),
        )
        .label("rn"),
    )
    .where(DailyPrice.date >= func.current_date() - 7)
    .subquery("ranked")
)
```

This uses `func.current_date() - 7` which SQLAlchemy translates to `CURRENT_DATE - 7` in PostgreSQL (integer subtraction = days). This is equivalent to `CURRENT_DATE - INTERVAL '7 days'` but simpler in SQLAlchemy.

The `func` import already exists at line 6 (`from sqlalchemy import select, func, literal_column, text`).

No other imports needed. No other changes to the file.

Why 7 days: We only need the latest 2 prices per ticker (rn=1 and rn=2). 7 days provides ample buffer for weekends and holidays while reducing the scan from 200K+ rows to ~2,800 rows (400 tickers × 7 days max).
